PRDの学術調長文を文章品質の機械強制で解消する
解決事例

AIが書くPRD（Product Requirements Document。製品要求仕様書）や設計書が学術論文調の冗長長文になる課題を、検査エンジン・機械強制・行動規約の3層で解消した事例。

§1概要

AIにPRDや設計書を書かせると、学術論文のような冗長な長文になり、カタカナ語も多くなる。現場では読まれず、結局書き直しになる。この課題を、検査エンジン・機械強制・行動規約の3層構造で解消した。都度の指示ではなく、成果物が確定する瞬間に機械で検査する仕組みを組み込んだ点が要点である。

導入は推進側が実施する。導入を進める場合は、文章品質の自動検査を入れたい旨を推進側へ伝えるだけでよい。本HTMLは課題・解決の型・導入の流れを説明する資料である。

§2課題

PRDや設計書をAIに書かせると、学術論文調の冗長な長文になりやすい。一文が長く、網羅性を優先するあまり同じ内容を言い換えて繰り返す。カタカナビジネス用語（ベストプラクティス・ステークホルダー・オンボーディング等）も多用され、初見の読み手には意味が伝わりにくい。現場担当者はこうした文章を読まず、結果として書き直しが発生する。

§3原因

生成AIの既定の文体は、網羅的・冗長で、一文が長く、受動態や外来語を多用する方向に寄りやすい。プロンプトで都度「簡潔に」「平易に」と指示しても、指示漏れや会話が長くなるうちの文体劣化で再発する。

要点
都度の指示だけでは再発を防げない。成果物が確定する瞬間（コミット・PR作成等）に機械で検査する仕組みが必要になる。

§4解決の型

検査エンジン・機械強制・行動規約の3層を積み上げて機能させる。どの層か1つだけでは効果が続かない。

§5実装の実物

読点一文につき3つまで
連続漢字8文字まで
その他の判定冗長表現の禁止・二重助詞の禁止・形式名詞のひらがな化

語彙64件・正規表現99パターンを6分類で管理する。置き換え例は次のとおり。

hookの発火点と強度

発火点検査対象強度
git commitドキュメントの追加・変更行のみ違反があれば停止

検査は追加・変更行のみが対象になるため、導入時に既存の過去文書を一括修正する必要はない。

§6効果

成果物が確定する経路（コミット・PR作成）のすべてに検問を置くことで、学術論文調の長文・カタカナビジネス用語・冗長な言い回しが成果物に残らなくなる。書き手がAIであっても人であっても、注意力に依存しない仕組みになっている。

§7実現するAI化レベル

検査は commit・PR 作成を契機に自動で走り（自動起動）、例外＝違反の条件は textlint ルールと置き換え辞書に列挙済みである。違反時だけ書き手が修正に入り、正常分は人が確認せずに通過する。

到達レベル判断根拠（誰が・何を確認する運用か）
レベル4（例外のみ人介入）相当検査は自動起動し、例外＝違反の条件が列挙済みである。違反時だけ書き手が修正に入り、正常分は人が結果を見ずに流れる運用であり、docs/02_AI化レベル基準の「検証主体の注記」における「人が結果を見ずに流れる運用」に該当する。

§8顧客環境への導入手順

導入は推進側が実施する。導入を進める場合は、文章品質の自動検査を入れたい旨を推進側へ伝えるだけでよい。

段階スキルが行うこと
①プロジェクト解析文書配置・Node.js環境・既存lint設定・既存hookの構成を調べる
②導入計画の提示と承認調査結果に基づく導入計画を提示し、実行前に承認を得る
③検査エンジンの導入textlint一式と置き換え辞書の雛形を配置する
④文体規約の配置常時読み込みの文体規約を配置する
⑤コミット検査hookの設置成果物確定の瞬間に違反を検知するhookを設置する
⑥実機検証違反語を含む試験コミットが停止することを確認する
⑦顧客語彙の初期登録ヒアリングで得た顧客固有の語彙を辞書に登録する

辞書は一般語彙のみの雛形から始め、顧客固有の語彙は導入後のヒアリングで追加する。コンサル側環境の辞書・設定の実物は使わない。

§9適用時の注意

AI文体検出プリセットは不採用
AI文体を検出する既定プリセット（ai-writing）は、定型の箇条書きまで誤検知するため採用しなかった。実測では30件のPRで119件が発火し、6割超が誤検知だった。
  • 検査対象は追加・変更行のみとし、既存資産は壊さない
  • 設定ファイル自体やAI実行ルール自体は検査対象から除外する

§10導入手段

本事例は資材を同梱しない。コンサル環境の設定・辞書の実物を顧客へ渡すと、環境固有情報の漏洩になるためである。導入に必要な雛形（検査設定・辞書雛形・文体規約・hookスクリプト）は推進側が保有し、顧客環境に合わせて配置する。
