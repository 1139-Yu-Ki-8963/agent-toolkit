# 解決事例集

本ドキュメントは、各案件で出た汎用的な課題と、その解決事例をまとめた蓄積簿である。案件で課題が出たら、まず本表を引く。既存事例があれば、事例HTMLをそのまま顧客への説明に使う。該当する事例がなければ、解決後に本表へ事例を追加する。

## 事例一覧

課題キーは連番禁止。内容を要約した意味語で付ける（番号からは情報を得られないため）。「実現レベル」は `docs/02_AI化レベル基準/AI化レベル基準.md` の基準で判定したL1〜L5。各事例の判断根拠は事例HTMLの「実現するAI化レベル」節に記載する。

## 事例の追加ルール

- 事例は `09_解決事例集/<課題キー>/` フォルダに、説明HTML（`解決事例.html`）を置く。環境構築を伴う解決策は導入スキルとして実装し、HTMLからスキル名を案内する
- 事例には「課題・原因・解決の型・実装の実物・導入手順・適用時の注意」を必ず含める
- 実装の実物は検証済みの設定・数値だけを書く（推測の効果・数値を書かない）
- このPC固有の絶対パスは事例本文に書かない（導入手順は顧客環境で再現できる書き方にする）
- 登録時は本表に1行追加する
- コンサル環境の設定・辞書の実物を事例フォルダに置かない（環境固有情報の漏洩防止）。同梱してよいのは顧客に渡す前提で作った一般資材のみ
