## タスク限定の観点（更新時に再検査する）

- Permission / Sandbox / WSL の3層構造が正確であること。Permission はツール呼び出し制御、Sandbox は Bash のOS レベル制限、WSL は実行環境（≠セキュリティ）

- Permission の具体例（Write→ask, Bash(rm -rf *)→deny, Read→allow）が仕様と整合していること
- Sandbox の OS 実装（macOS→Seatbelt, Linux・WSL2→bubblewrap）が正確であること
- WSL の説明に「サンドボックスではない」「土台として動く」が明記されていること
- 層の色分け: セキュリティ層=緑系（--shield）、実行環境=グレー系（--platform）
