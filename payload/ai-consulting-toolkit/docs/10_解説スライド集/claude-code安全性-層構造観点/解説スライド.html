開発ツール運用ガイド

Claude Code の安全性は 2 層 + 実行環境で成り立つ

Permission はツール呼び出しを、Sandbox は Bash のシステムアクセスを制御する。WSL はサンドボックスではなく実行環境。

1 Permission セキュリティ層
Claude のツール呼び出しを制御する Read・Write・Edit・Bash 等の各ツールに対し、allow（自動許可）/ ask（都度確認）/ deny（禁止）の 3 段階で制御する。deny は他の設定で上書きできない。
Write → ask Bash(rm -rf *) → deny Read → allow
実装: Claude Code 内部の権限エンジン（settings.json で設定）
2 Sandbox セキュリティ層
Bash コマンドのファイル・ネットワークを OS レベルで制限する 書き込み可能なディレクトリをホワイトリストで限定し、ネットワーク接続先を許可ドメインに制限する。Permission を通過した Bash コマンドにさらに適用される。
書込先をプロジェクト内に限定 接続先ドメインを許可制に
実装: macOS → Seatbelt / Linux・WSL2 → bubblewrap
↑ セキュリティ層　──　↓ 実行環境
3 WSL 実行環境
Windows 上で Linux カーネル環境を提供する（≠ セキュリティ） Claude Code が動作する土台。WSL 自体はアクセス制限を行わない。WSL2 の中で Sandbox（bubblewrap）が別途動作し、ファイル・ネットワークを制限する。
仮想化された Linux 環境 macOS / Linux では不要
実装: Microsoft WSL2（Hyper-V ベースの軽量仮想化）
