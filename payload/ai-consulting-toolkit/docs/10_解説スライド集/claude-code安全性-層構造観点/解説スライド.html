開発ツール運用ガイド

Claude Code は 2 層で安全性を確保する

Permission がツール呼び出しを制御し、Sandbox が Bash のシステムアクセスを OS レベルで制限する。

1 Permission ツール制御
Claude のツール呼び出しを allow / ask / deny で制御する Read・Write・Edit・Bash 等の各ツールに対し、自動許可（allow）/ 都度確認（ask）/ 禁止（deny）の 3 段階で設定する。deny は他の設定で上書きできない最強の制限。settings.json で管理する。
Write → ask（都度確認） Bash(rm -rf *) → deny（禁止） Read → allow（自動許可）
Permission を通過した Bash コマンドに、さらに Sandbox が適用される
2 Sandbox OS 制限
Bash コマンドのファイル書き込みとネットワーク接続を OS レベルで制限する 書き込み可能なディレクトリをホワイトリストで限定し、ネットワーク接続先を許可ドメインに制限する。Claude Code のアプリケーション層ではなく、OS のセキュリティ機構を使って強制する。
書込先をプロジェクト内に限定 接続先ドメインを許可制に 認証情報ファイルの読取を遮断
OS 実装: macOS → Seatbelt / Linux → bubblewrap
1

Permission— Claude が使えるツールと操作を制御する。危険な操作は deny で完全に禁止できる

2

Sandbox— Permission を通過した Bash コマンドに対し、OS レベルでファイル・ネットワークを制限する

2026年7月時点の Claude Code 公式仕様に基づく。
