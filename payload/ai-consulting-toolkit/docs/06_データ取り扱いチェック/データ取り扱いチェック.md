# データ取り扱いチェック

顧客データをAIサービスに入力する前に、必ず通す判定である。ヒアリングから効果測定まで、業務6フェーズのどの段階でも横断で適用する。

## チェック項目

| 項目キー | 確認内容 | 確認先 |
|---|---|---|
| 個人情報有無 | 氏名・連絡先など、個人を識別できる情報を含むか | データ提供元（顧客担当者）への確認、または元データの項目一覧 |
| 機密区分 | 顧客社内の情報区分で「秘」「極秘」に当たるか | 顧客の情報管理規程、または案件責任者への確認 |
| 契約制約 | 業務委託契約・秘密保持契約にAI利用を制限する条項があるか | 契約書本文（秘密保持条項・再委託条項） |
| 学習利用設定 | 利用するAIサービスが入力データを学習に使わない設定になっているか | 各AIサービスの管理画面設定、利用規約 |
| 保存先所在地 | データの保存先が契約で許された国・地域か | AIサービス提供事業者が公開するデータセンター所在地情報 |
| 再識別危険 | 伏せ字化した後も、他の情報と組み合わせて個人や取引を特定できないか | 加工後データを第三者の目線で見直す確認 |

## 判定フロー

以下の分岐で「そのまま渡してよい」「加工すれば渡してよい」「渡してはならない（ローカル処理のみ）」の3値を判定する。

```
                    入力予定データ
                         |
                         v
          個人情報・機密情報を含むか？
              |                  |
            いいえ                はい
              |                  |
              v                  v
     そのまま渡してよい   契約制約・学習利用設定・保存先所在地の
                          3項目をすべて満たすか？
                              |              |
                             はい            いいえ
                              |              |
                              v              v
                  伏せ字化で再識別危険を   渡してはならない
                  消せるか？             （ローカル処理のみ）
                      |         |
                     はい       いいえ
                      |         |
                      v         v
          加工すれば渡してよい   渡してはならない
                                （ローカル処理のみ）
```

「加工すれば渡してよい」とは、固有名詞・数値を伏せ字に置き換える加工（以下、伏せ字化）を施した上でAIサービスに渡すことを指す。伏せ字化の具体例は次章を参照する。

## 加工の実務

伏せ字化の具体例を示す。

**例1: 固有名詞の伏せ字化**

- 加工前: 「株式会社○○の田中様からご相談があった件」
- 加工後: 「A社のご担当者からご相談があった件」

**例2: 金額・口座番号の伏せ字化**

- 加工前: 「口座番号1234567へ振込額3,200,000円を送金」
- 加工後: 「口座番号XXXXXXXへ振込額X,XXX,XXX円を送金」

案件フォルダに判定結果を記録として残す。表の雛形は次の通り。

| 対象データキー | 判定日 | 判定結果 | 加工内容 | 判定者 |
|---|---|---|---|---|
| （記入例）議事録-顧客名 | 2026-07-08 | 加工すれば渡してよい | 顧客名・担当者名を伏せ字化 | 山田 |
