# 業務棚卸しテンプレート

本表は**事実の記録簿**である。推測・解析による記入を禁止する。段階3（棚卸し）から段階4（レベル判定）にかけて使用する。判定レベルは到達可能上限（マトリクス適用結果）を示す。現在レベルは consult-hearing の現在地診断に基づいて記入し、目標レベルは consult-level-agree での顧客合意後に記入する。昇格時は consult-measure が実績を根拠に更新する。

## 記入表

| タスクキー | タスク名 | 担当 | 誤り影響度 | 判断定型度 | 入力構造化度 | 検証容易性 | 判定レベル | 現在レベル | 目標レベル | 根拠・出所 | 備考 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| | | | | | | | | | | | |

- 時間・頻度・件数の推定値を本表に記入しない。時間の定量は `../05_効果測定テンプレート/効果測定テンプレート.md` の導入前実測で扱う。
- 4軸の評価は担当者への質問で得た回答の記録に限る。AI・記入者の推測による評価を禁止する。回答が得られない軸は「未確認」と記入する。
- 「根拠・出所」には誰から・いつ・何で確認したか（発言・提示資料・実演のいずれか）を書く。
- 判定レベルは到達可能上限であり、`../02_AI化レベル基準/AI化レベル基準.md` の判定マトリクスを未確認軸のない行にだけ適用した結果である。未確認軸が残る行は「保留」と記入する。
- 現在レベルは consult-hearing の現在地診断（`../02_AI化レベル基準/AI化レベル基準.md` の診断の質問・証拠原則）に基づいて記入する。まだAIを使っていなければレベル1（人手のみ）。昇格時は consult-measure が実績を根拠に更新する。
- 目標レベルは今回の案件で到達を合意したレベルを記入する。判定レベル（上限）の範囲内で段階導入の原則に従う。consult-level-agree で合意するまでは空欄とする。
- タスクキーは連番禁止。内容を要約した意味語で付ける（番号からは情報を得られないため）。

## 記入手順

1. ヒアリングの課題サマリーから、対象業務に含まれるタスクを列挙する
2. 1タスクが1つの完結した作業単位になるよう分割する
3. 4軸（誤り影響度・判断定型度・入力構造化度・検証容易性）を担当者に質問し、回答をそのまま記録する。推測で埋めない
4. 質問の根拠・出所（誰から・いつ・何で確認したか）を記録する
5. 未確認軸のない行に `../02_AI化レベル基準/AI化レベル基準.md` の判定マトリクスを適用する。未確認軸が残る行は「保留」とし、追加確認の予定を備考に書く

## 次のフェーズへの引き継ぎ

判定レベルが付いた行のみ `../04_レベル別タスクテンプレート/レベル別タスクテンプレート.md` に渡し、タスクごとにWBS（Work Breakdown Structure。作業を階層分解した一覧表）へ展開する。
