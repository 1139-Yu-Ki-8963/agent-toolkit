# 変更のレビュー・テスト（review-and-test）

実装・修正が完了した変更を、規約の観点表と照合してからテストを実行し、全項目が通るまでを1つの流れで完了させる。

## Phase 1: 変更差分の確認

`git diff` で変更対象ファイルを特定する。対象外のファイルが含まれていないか確認する。

## Phase 2: 規約観点レビュー

`{{RULE_PATH}}` の規約から生成した観点表と、変更差分を照合する。レビューは変更を書いた本人ではなく、新規サブエージェントに依頼する。

観点表:

| 観点キー | 確認内容 | 判定 |
|---|---|---|
| 命名-規約準拠 | 命名規則に沿っているか | |
| 構成-配置単位 | ファイル分割・配置が規約の単位と一致するか | |
| エラー処理-規約準拠 | エラー処理が規約の書き方と一致するか | |
| テスト-配置準拠 | テストの配置・命名が規約と一致するか | |
| 禁止事項-非該当 | 禁止事項に該当する記述がないか | |

## Phase 3: テスト実行

`{{TEST_COMMAND}}` を実行し、結果を記録する。

## Phase 4: 修正ループ

Phase 2 の指摘、または Phase 3 の失敗があれば修正し、Phase 2 に戻る。上限は5回。同一の指摘・同一のエラーが2回連続したらループを止め、上位に差し戻して報告する。

## Phase 5: 報告

観点表の判定一覧とテスト結果を報告する。全項目 PASS であればコミット可と伝える。
