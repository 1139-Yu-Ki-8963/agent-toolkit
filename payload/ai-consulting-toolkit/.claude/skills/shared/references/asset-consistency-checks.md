# スキル資産間の整合性検査仕様

改善指示書の反映後、スキル資産間（テンプレート・スキーマ注記・SKILL.md・基準文書）の整合性を機械的に検査する仕様。

## 検査項目

### 既定値算出式の一致

consult-hearing の開始レベル提案（SKILL.md Phase 7）と consult-level-agree の goalLevel 既定値（SKILL.md + goal-agreement-procedure.md + テンプレートのスキーマ注記）が同一の式 `min(現在レベル+1, 判定レベル)` を参照しているか。

検査方法: 両スキルの SKILL.md と関連ファイルを grep し、既定値に関する記述が「判定レベルそのもの」ではなく「min(現在レベル+1, 判定レベル)」になっていることを確認する。

### タグ語彙の閉包

level-agree-template.html のスキーマ注記のタグ語彙リスト（行頭タグとして定義された語彙）と、JS 内のフィルタ（isNonToggle 関数、引き継ぎ判定）が整合しているか。引き継ぎ判定が語彙リストに無いタグを条件に使っていないことを確認する。

検査方法: テンプレートのスキーマ注記から行頭タグの語彙リストを抽出し、JS 内の正規表現フィルタと突合する。

### 表示文言の自己完結

テンプレート HTML のパネル見出し（renderPanel 関数内の panelSec 呼び出しと el("h3", ...) 呼び出し）に指示語パターン（「このレベル」等）が残っていないか。

検査方法: テンプレートの JS 内で `panelSec(` と `el("h3"` を含む行を grep し、「このレベル」「そのレベル」等の指示語が0件であることを確認する。

### 前提定型文の一致

consult-level-agree/SKILL.md の前提記述規則（premises の整理手順）とテンプレートのスキーマ注記の前提データ仕様が一致しているか。「1件目は必ず定型文」という旧式の記述が残っていないことを確認する。

検査方法: テンプレートのスキーマ注記と SKILL.md の前提関連記述を grep し、「必ず定型文」が0件であることを確認する。
