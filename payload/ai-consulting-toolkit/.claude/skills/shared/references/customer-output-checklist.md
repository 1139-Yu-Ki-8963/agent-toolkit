# 顧客向け出力の自問チェック（共通）

顧客に渡す・見せる出力（HTML・Excel・依頼文書・報告表）を出す前に、次を全て確認する。
1つでも該当したら生成元の Phase に戻って修正し、再チェックする。

- consult- で始まる内部スキル名を含めていないか
- 特定PC・コンサル環境固有のパスを含めていないか
- 作業用の内部分類・記号見出し（①②③・(a)(b)(c)・判定ラベル・状態語）をそのまま出していないか
- 根拠のない数値・頻度・時刻を断定していないか
- 課題・項目の名称が顧客に提示済みの名称と一致しているか（内部の課題キーをそのまま出していないか）
- 内部の中間生産物（受付簿・下書き・台帳・管理データ）を成果物として列挙していないか（成果物は顧客が受け取る・目にする資料に限る）
- パネル見出し・セクション見出しに、文外参照の指示語（「このレベル」「その課題」等）を使っていないか（参照先の固有名を差し込むこと）
- 内部の記録ファイル名（拡張子付きのファイル名等。例: 課題受付簿.md、WBS.json）が表示テキストに出ていないか
- 内部の管理語彙（管理フォルダ・記録が無い・正本・転記等の内部運用表現）が表示テキストに出ていないか

各スキル固有の追加チェックは、各スキルの SKILL.md の出力前チェック節に列挙する。

本ファイルが共通チェックの正本であり、項目の追加・変更はここで行う。
