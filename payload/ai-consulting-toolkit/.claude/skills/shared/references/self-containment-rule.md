# 自己完結原則

顧客提出資料の文は単独で意味が確定すること。文をまたぐ参照・画面状態への参照を指示語で行わず、参照先の固有名（レベル番号・課題名・基準名等）で書く。

## 対象外（指示語を使ってよい場合）

- 同一文内で直前の名詞を受ける用法
- 顧客発言の引用原文
- 熟語化した語（その他・それぞれ・そのまま等）
- 疑問語（どの・どれ・どこ）
- 直後に参照先のリストが続く「以下の」

## 検出パターン

エラー（見出し・カード・表セル・箇条書き先頭で検出した場合）:
- (この|その|あの)(レベル|課題|基準|段階|仕組み|作業|資料|一覧|文書|現場|場合|方法)
- (これ|それ|あれ)(は|が|を|により|以降|以外)
- (上記|下記|前述|先述|後述)

警告（本文で検出した場合）: 同上のパターン

除外: その他／それぞれ／そのまま／どの／どれ／どこ／引用ブロック内／コードブロック内／「以下の」のうち同一節内で参照先のリストが直後に続くもの

## 検査対象

- HTML: タグ・スクリプト・スタイルを除去した後の表示テキスト
- Markdown: 本文

検出パターンと除外リストは本ファイルを single source of truth とし、Markdown用・HTML用の両検査から参照する（二重管理の禁止）。
