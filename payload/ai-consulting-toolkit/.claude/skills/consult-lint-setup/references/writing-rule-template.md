# 文章品質規約

このプロジェクトのドキュメントは、読み手が一読で意味を取れる文章であることを最優先とする。以下の規約に反する文は、機械強制（textlint + 置き換え辞書）で検出し修正する。

## 文の骨格

- 修飾節は被修飾語の直前に置く。長い場合は文頭に移動するか別文にする
- 一文は100文字以内に収める。超える場合は文を分割する
- 主語と述語の間に長い挿入句を入れない。挿入句が長くなる場合は文を分割する

## 表現

- 受動態（〜される・〜られる）は能動表現に変える
- 二重否定（〜ないわけではない等）を使わない
- 冗長表現を避ける。次の言い換え例に従う
  - 「〜することができる」→「〜できる」
  - 「〜という状況である」→「〜である」
  - 「〜を行う」→ 具体的な動詞（「実行する」「実施する」等の空虚な代動詞を避ける）

## 語彙

- カタカナビジネス用語は置き換え辞書（prh.yml）の推奨語に従う
- 略称・頭字語は初出時に正式名称と1行説明を添える

## 機械強制

この規約の語彙・文長は textlint + prh 辞書がコミット時に検査する。

- 検査設定: `tools/linter/.textlintrc.json`
- 置き換え辞書: `tools/linter/prh.yml`
- コミット検査 hook: `.claude/hooks/check-textlint-commit.sh`
