# <機能名> 単体テスト観点表

本書は関数・メソッド・ドメインロジック単位（small）の「何を検証すべきか（観点）」の正である。TDD 二重ループの**内側ループ**（red → green → refactor）の RED 起点として使う。具体的なテストケース（入力値・アサーション）は管理しない — テストコードが正。

## テストサイズ対応表

| サイズ | 実行範囲 | 例 | 置き場 |
|---|---|---|---|
| small | 1 プロセス内で完結 | vitest・関数単位テスト | 単体テスト観点表（+ 結合表の一部） |
| medium | 1 マシン内（実ブラウザ・ローカル実 DB 可） | API 統合テスト | 結合テスト観点表 |
| large | 複数システム・実環境・画面横断 | 画面をまたぐユーザージャーニー | 03_操作フロー設計 |

所属は実行技術ではなく、対象がドメインロジック単体に閉じるかで決める。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 具体的な入力値・手順・アサーション | テストコード |
| ドメインをまたぐ結合・DB 整合・API 契約（medium）・受け入れ条件 | 結合テスト観点表 |
| 画面をまたぐシナリオ（large） | 03_操作フロー設計 |
| 画面の UI 検証 | 02_画面基本設計 側の単体テスト観点表 |
| lint・静的解析 | Phase 7 完了チェック / hooks |

> 対象は実装前なので業務粒度の名前で書く（関数名はまだ存在しない）。実装後にテスト参照列で実体と結ぶ。
> キーは連番禁止。内容を要約した意味語で付ける（番号からは情報を得られないため）。

| キー | 対象 | 観点 | 由来 § | 区分 | テスト参照（実装後） |

## TDD 内側ループ規則

- 本書は TDD 二重ループの**内側ループ**を担う（外側ループは結合テスト観点表の受け入れ ◯ 観点）
- 観点表を上から順に RED（テスト先行）で書く
- 1 行 = 1 つ以上のテストに対応する
- GREEN 後にテスト参照列を記入する
- 実装中に新観点を発見したら行を追加してから RED を書く（テスト先行を維持する）
- 全行にテスト参照が付いたら frontmatter の `status: traced` に更新する

## 自動化宣言

単体（small）は全観点を自動化する。自動化できない観点が出た場合は結合テスト観点表の手動観点へ移す。本書は手動観点の枠を持たない。
