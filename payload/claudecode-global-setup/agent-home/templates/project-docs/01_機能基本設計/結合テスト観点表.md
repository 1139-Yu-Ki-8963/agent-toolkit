# <機能名> 結合テスト観点表

本書は機能内の結合（複合ルール・API 契約・DB 整合・権限ポリシー）と中規模統合検証（medium）の観点の正であり、**この機能の受け入れ条件の正**でもある。自動テストの具体的な手順・アサーションはテストコードが正。手動観点のみ本書が手順を持つ。

## テストサイズ対応表

| サイズ | 実行範囲 | 例 | 置き場 |
|---|---|---|---|
| small | 1 プロセス内で完結 | vitest・関数単位テスト | 単体テスト観点表（+ 結合表の一部） |
| medium | 1 マシン内（実ブラウザ・ローカル実 DB 可） | API 統合テスト | 結合テスト観点表 |
| large | 複数システム・実環境・画面横断 | 画面をまたぐユーザージャーニー | 03_操作フロー設計 |

所属は実行技術ではなく、対象が 1 機能内に閉じるかで決める。

## 二重ループ・受け入れ規則

- 受け入れ ◯ の観点は **Phase 6 開始時に失敗するテストとして先行作成する（TDD 二重ループの外側ループ）**
- 内側ループ（単体テスト観点表）の消化が進むことで外側ループが green に近づく
- 受け入れ ◯ が全て green（自動）または確認済み（手動）になったとき機能完成とみなす
- 受け入れ ◯ は「この機能がリリース可能」と業務語で言い切れる観点に限る
- 受け入れ ◯ の全 green と手動観点の確認済みが PR マージの条件
- 全行にテスト参照または手動参照が付いたら frontmatter の `status: traced` に更新する
