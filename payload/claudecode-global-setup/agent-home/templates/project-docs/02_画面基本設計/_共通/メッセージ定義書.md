# プロジェクト共通メッセージ定義書

本書はプロジェクト全体の UI 文言の **正（権威ある仕様源）** である。
各画面設計書の §4（画面状態）・§5（入力と検査）に対応する文言は、個別に書かずにここのキーを参照する。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 文言の表示実装・出し分けロジック | 実装コード |
| 画面固有の一回きり文言（定型化しないもの） | 当該画面設計書の §4 または §5 |
| 数値・色・フォント | _共通/DESIGN.md |

## 記入規則

- キーは **意味キー** で付ける（例: `保存成功-トースト`・`権限なし-ダイアログ`・`通信失敗-リトライ案内`）
- 連番禁止（`MSG-001` のような識別子は使わない）
- 同じ文言が複数画面で使われる場合のみ本書に収録する。1 画面限定の文言は当該画面設計書に置く
- 文言は断定調・敬体で統一する（プロジェクト方針に合わせて変更可）

| キー | 文言 | 種別 | トーン・用途 | 使用画面 |

| 通信失敗-リトライ案内 | <通信に失敗しました。しばらく待ってから再試行してください> | インライン | エラー・ネットワーク障害時 | <全画面共通> |
| 入力必須-インライン | <<項目名> は必須です> | インライン | バリデーション | <入力フォームを持つ画面> |
| 読込中-スケルトン | — | スケルトン | ローディング・テキスト代替 | <全画面共通> |

## approved の条件

- 未決定の文言がゼロ件であること
- textlint 検査・リンク検査（lychee）が PASS であること
