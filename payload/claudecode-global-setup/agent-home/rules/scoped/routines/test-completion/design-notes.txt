# ルーティンテスト完了判定規約（ROUTINE-TEST-COMPLETION）設計ノート（非注入サイドカー）
# rule.md から分離した保守者向け記録。規約の挙動には影響しない。

## 背景

2026-06-21〜22 のルーティンテストセッションで、以下の違反を繰り返した:

- JSONL フィールド名の一致だけで PASS（Layer 3 未実施）
- 1 回のみの実行で完了宣言（2 連続なし）
- skipped_no_date 不一致・自己参照誤検出を「軽微」として無視
- ユーザーに指摘されるまで問題に気づかなかった（4 回指摘された）

## 設計判断

**必要性**: Claude は長時間セッションで「進捗を見せたい」圧力に負けて合格基準を甘くする傾向がある。testing.md に基準を書いても、宣言時にチェックしない。宣言自体を規制する rules が必要。

**代替案を採用しなかった理由**:
- testing.md に追記: 既に基準は書いてあるのに守らなかった。基準の場所ではなく宣言の制御が必要
- CLAUDE.md に追記: 機械強制が必要なものは rules に閉じる方針

**保守責任者**: ユーザー

**廃棄条件**: Claude Code 本体がルーティンテストの合否判定を自動化した時

2026-07-11: agent-home ローカルからグローバルツリーへ移設。**/routines/** は理論上他リポジトリの同名ディレクトリにマッチしうるが、routines ディレクトリは agent-home にしか存在しない運用のため許容（ユーザー承認済み）。
