# ルーティンテスト完了判定規約（ROUTINE-TEST-COMPLETION）

ルーティンテストで「PASS」「完了」「○」を宣言する前に満たすべき必須条件を定義する。

## 完了宣言の禁止条件

以下のいずれかに該当する場合、PASS / 完了 / ○ を宣言してはならない。

1. **2 連続 PASS を取得していない** — testing.md の収束条件は「Layer 1-3 全 ○ が 2 連続」。1 回の実行で PASS にするのは禁止
2. **Layer 3 サブエージェント調査を実行していない** — JSONL のフィールド確認だけで PASS にするのは禁止。PR diff の品質・計測値の裏取り・成果物の中身の調査を実際に実行した証跡がなければ PASS ではない
3. **残課題を「軽微」と矮小化している** — 検証仕様の合格基準に × がある場合、件数や深刻度に関係なく FAIL。「軽微だから ○」は存在しない
4. **早期終了で未検証の Phase がある** — 早期終了は Phase 1 の検証のみ。Phase 2-6 のスキーマ・行動・忠実度はフル実行の JSONL で別途検証が必要

## 完了宣言に必要な証跡

PASS を宣言するとき、以下をユーザーに提示すること:

```
## PASS 証跡
- run_id（1 回目）: <id>
- run_id（2 回目）: <id>
- Layer 3 エージェント ID: <id>（調査結果サマリを添付）
- 残課題: なし（× が 1 件もないことを明示）
```

証跡なしの PASS 宣言は無効とする。

## プロジェクト上書き

- 上書き可否: 一律適用
- 理由: ルーティンテストの完了基準はルーティン共通の枠組みであり、プロジェクトに依存しないため受け口を設けない

設計判断・経緯の記録は同ディレクトリの `design-notes.txt` を参照（非注入サイドカー）。
