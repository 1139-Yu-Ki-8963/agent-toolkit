# 全グローバル rule のレビュー照合分類（非注入・監査専用データ）
#
# 所有者: 同ディレクトリの rule.md（レビュー観点フォルダ統治規約）
#
# 「レビュー観点はどの rule にあり、どれをレビューで照合し、どれを照合しないか」の正本。
# audit-review-coverage.sh が、ディスク上の全 rule.md（always/ + scoped/）が
# 「review-checklist 配下（scoped = 専門家別観点 / always = 全専門家が照合する横断観点）」
# または「本ファイルに分類記載」のどちらかに該当することを検証する。
# 未分類の rule が 1 件でもあれば監査 FAIL となる（新規 rule 追加時は必ずここへ分類を追記する）。
#
# 分類の定義:
#   config-asset    … 設定資産（skills/rules/hooks/routines/subagents）の規約。
#                     レビューは managing-agent-configs（create→review→test→マーカー→commit gate の
#                     ライフサイクル一体型）が担当し、reviewing-against-rules の対象外とする。
#                     観点が references にあるのは意図的（ゲート機構と観点が一体のため）
#   hook-enforced   … hook・permissions が行為の瞬間（commit/push/Write/起動時）に機械強制する
#                     プロセス・行為規約。成果物の内容観点ではないため、レビューで二重照合しない
#   behavior        … エージェントの行動規範（常時注入が防衛線）。成果物の内容観点ではない
#
# レビュー観点そのものはここに載せない。場所が分類になる:
#   scoped/review-checklist/<domain>/<観点>/ … <domain>-reviewer が担当する観点
#   always/review-checklist/<観点>/          … 常時注入かつ全レビュー専門家が横断照合する観点
#                                              （用語辞書・連番ID・略称。旧 always-rules.txt は廃止）
#
# 形式: <rule.mdの~/形式パス><TAB><分類><TAB><補足>
