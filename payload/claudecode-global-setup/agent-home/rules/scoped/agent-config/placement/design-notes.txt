# rule.md から分離した保守者向け記録。規約の挙動には影響しない。

## 設計判断

**必要性**: Claude が設定・規約・手順を書く際に「どの層に置くか」を誤る事例が繰り返し観測された。CLAUDE.md に全部書く / rules に手順書を書く / hook にプロンプトを埋め込む等のアンチパターンを、決定木として明示することで配置判断を構造化する。

**代替案を採用しなかった理由**:
- CLAUDE.md に決定木を書く: CLAUDE.md 自体が 200 行制限を持ち、決定木（82 行）を収容すると他の内容を圧迫する
- Skill 化: 配置判断は「特定トリガーで起動する手順」ではなく「常時参照する判断基準」。lazy rule として該当ファイル操作時に注入するのが適切

**保守責任者**: 人手（ユーザー）。新しい設定層が追加された時に決定木を更新する。

**廃棄条件**: Claude Code 本体が設定層の配置先を自動判定する機能を組み込んだ時。

## scoped 配置の理由（2026-07-06 追記）

配置決定木は設定資産（CLAUDE.md・.claude/・SKILL.md）を作成・編集する場面でのみ必要な判断基準であり、通常の実装・調査タスクでは参照されない。paths 条件付き注入で該当作業時のみロードする。
