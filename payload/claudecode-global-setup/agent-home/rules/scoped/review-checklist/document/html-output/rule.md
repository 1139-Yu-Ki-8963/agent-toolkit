# 単一HTML成果物規約（SINGLE-HTML-OUTPUT）

文書・説明資料としての単一HTML成果物を作成する際の観点規約。入力内容をただHTMLへ変換する汎用HTML化を禁止し、対象コンテキストに対する理想出力の推論・図解候補の全件調査・合格基準の充足を義務付ける。

## 適用範囲

- 対象: 文書・説明資料・レポートなど、自己完結する単一HTMLファイルとして完結する成果物の新規作成・改修
- 対象外: アプリケーションのテンプレート・コンポーネントとしてのHTML（フレームワークの一部として動作するもの）

## 基本原則

- 入力内容をただHTMLへ変換しない。あらかじめ決められた見た目や構成に流し込まない
- 既存の典型的な表現方法に当てはめるのではなく、対象コンテキストに対して望ましい理想出力を先に考え、その理想を単一HTMLでどこまで実現できるかを検討する
- 文章だけで意味を提示し、図解可能な関係を未検討のまま残すことを禁止する

一般解の禁止には、少なくとも次の行為の禁止を含む。

- コンテキストの性質と無関係な既製テンプレートの装着
- 性質が異なる要素群への単一既製コンポーネントの一律適用（対象側の根拠記録なし）
- 性質が異なる図解候補への単一図解形式の一律適用
- 理想出力の検討を経ないままの生成着手
- 図解候補の抽出を行わないままの生成着手

## 図解候補の全件調査

対象コンテキストに含まれる次の性質を全件調査し、図解候補として扱う。

- 手順、工程、時系列
- 階層、包含、分類
- 対応関係、依存関係、接続関係
- 比較、対比、差分
- 原因、結果、条件、分岐
- 状態および状態遷移
- 入力、処理、出力
- 全体と部分
- 数量、割合、増減、分布
- 配置、位置、範囲
- 複数要素間のその他の意味的関係

図解によって理解速度・全体把握・関係把握・記憶可能性が向上する候補は図解する。図解を採用しない候補には、対象コンテキストに基づく不採用理由を記録する。

## 構造・図解の採用評価（5項目）

コンポーネントおよび図解は、単体であっても最低限、次の評価項目を満たす。

1. この構造または図解は、対象コンテキストのどの性質から導かれたか
2. 既存コンポーネントまたは既存の図解形式を使う場合、その採用理由は対象側にあるか
3. 読者の理解をどう助けるか
4. 代替構造または代替図解を棄却した理由は何か
5. 説明できない構造または図解は削除または再設計する

この5項目は頭の中で考えるだけでは足りない。採用した各構造・各図解について回答を記録する。回答を記録できない構造・図解の採用は不合格である。

## 設計領域（11領域）

HTML化の設計領域は次の11領域で閉じる。全領域が設計記録の対象であり、「その他」や「etc」による記載回避を認めない。

1. 情報設計（分節・階層・順序・グルーピング）
2. 構造化（セマンティックマークアップ、見出し階層、リスト・表・セクションの選定）
3. 全体的レイアウト設計
4. 局所的レイアウト設計
5. 視覚設計（タイポグラフィ、余白、強弱、密度）
6. カラーリング設計（色の役割定義、コントラスト）
7. 図解設計（図解候補の抽出、採否、図解形式、配置、情報密度、ラベル、文章との関係）
8. 読解導線（目次・ナビゲーション・視線誘導）
9. 操作設計（インタラクションの要否判断と仕様。不要と判断する場合も、その根拠を記録する）
10. 自己完結性（外部依存の排除）
11. 単一HTMLとしての成立性

## 図解形式の選択肢

図解形式は、表現対象となる意味に基づいて選択する。選択肢には、少なくとも次を含む。

フロー図 / 時系列図 / 階層図 / ツリー図 / 関係図 / 構成図 / 対応図 / 比較図 / マトリクス / 状態遷移図 / 因果図 / 入力・処理・出力図 / 全体・部分図 / 数量図 / チャート / 注釈付き模式図 / その他、対象固有の意味を表現する図解

## 図解の質

- 「図解を限界まで豊富にする」とは、図解数を機械的に増やすことではない。全図解候補について理解上の効果を検討し、効果が認められる候補をすべて実装することをいう
- 同一内容を形式だけ変えて反復した図、意味的関係を持たない装飾、色付きの箱を並べただけの構造、本文を分割して配置しただけの構造は、図解数に含めない
- 図解は本文の代替ではなく、本文に含まれる意味を別の認知経路から理解可能にする表現として設計する

## 合格基準（最低出力仕様）

最終成果物は単一HTMLである。以下の全項目に合格しない限り、出力を完成として扱わない。本基準は下限であり上限ではない。下限のみを狙った最小実装は、11設計領域の記録で対象コンテキストへの最適性を説明できない限り不合格とする。

### 被覆

- 入力コンテキストの全内容が、出力HTML内に対応箇所を持つ（被覆率100%）。欠落の判定単位は「最終HTMLから欠落したときに欠落と判定できる最小の意味まとまり」（見出し、段落、箇条書きの各項目、表の行、コード片、数値・固有名詞を含む主張、図示対象など）とする
- 省略表現（「以下略」「他N件」「残りは同様」、内容の代替としての三点リーダなど）で内容を代替しない
- 入力量の大きさは、被覆低下の理由として認めない

### 図解

- 全図解候補について、採用または不採用の判定を記録する（判定率100%）
- 理解速度・全体把握・関係把握・記憶可能性が向上する候補は、すべて出力HTML内に実装する
- 不採用とした候補には、対象コンテキストの性質に基づく具体的な不採用理由を記録する。「不要」「適さない」など根拠を示さない記載は無効とする
- 各図解の形式は、表現対象となる意味的関係に適合させる。形式の採用理由を説明できない図解は不合格とする
- 図解を、入力内容の省略・代替のために使用しない。図解を配置しても被覆の充足義務は残る
- 同一内容の反復、装飾のみの図形、意味的関係を持たないカード列、本文の視覚的分割のみを図解として計上しない
- 図解内の要素、線、矢印、軸、領域、色その他の符号には、意味を特定できるラベルまたは凡例を付す
- 図解は狭幅と広幅の双方で判読可能とし、文字・線・ラベル・要素が欠落、重複または不可読にならない
- 図解の内容を理解するために必要な情報を、図解内のラベル、説明文、キャプション、または同等のテキスト表現としてHTML内に保持する

### 禁止パターン

- プレースホルダの残置禁止（TODO、FIXME、lorem ipsum、「ここに〜が入ります」、サンプルテキストなど）
- 機能しない操作要素の禁止（処理が結線されていないボタン、遷移先のない見せかけリンク、実装されないままのUI）。操作不要と判断した場合は、操作要素自体を置かない
- 未閉タグ・生成途中打ち切りの痕跡の禁止
- コメントアウトによる内容退避の禁止（コメント内へ本文を隠して被覆を装う行為を含む）

### 構造

- 意味に基づくマークアップを行う。全文を単一のdivやpreへ流し込む構造、見出し階層の飛び・不整合を禁止する
- 文書メタを完備する（DOCTYPE、html要素のlang属性、文字コード指定、ビューポート指定、タイトル）
- 図解には、figure・figcaption・svgその他、その役割に適した構造を使用する。図解と無関係な要素の寄せ集めによって意味構造を偽装しない

### 体裁

- ファイル内styleによる、対象コンテキスト専用に設計されたCSSを持つ。ブラウザ既定表示のままの提出を禁止する
- 出力は設計記録と一致させる。記録にない見た目、記録と矛盾する見た目は不合格とする
- 狭幅（モバイル相当）と広幅（デスクトップ相当）の双方でレイアウトが破綻しない
- 図解と本文の視覚的優先順位、参照関係および配置関係を明確にする

### 自己完結

- 成果物は1ファイルであり、CSS・JavaScriptはファイル内に内包する
- 表示と機能がネットワーク到達不能環境で成立する。外部リソース（CDN、外部フォント、外部画像など）を表示・機能の必須条件にしない
- 図解に使用するSVG、画像相当表現、データ、スクリプトおよびスタイルはファイル内に内包する

## 機械強制

現時点では hook による機械強制なし。単一HTML成果物の作成・レビュー時に、本規約の合格基準を手動確認の観点として使う。

手動レビュー時の検出補助: `grep -nE 'TODO|FIXME|lorem ipsum' <file>.html`（禁止パターンの一部のみ。被覆・図解の検証は目視で行う）

## 違反検知時の手順

本規約は注入タグを発行しない。単一HTML成果物の作成中・レビュー中に合格基準への不適合を発見した場合:

1. 不適合箇所を合格基準の該当小見出し（被覆 / 図解 / 禁止パターン / 構造 / 体裁 / 自己完結）に対応付けて特定する
2. 修正が図解の追加・削除を伴う場合は、「構造・図解の採用評価（5項目）」に従い採否理由を記録し直す
3. 修正後、合格基準の全項目を再確認してから完成として扱う

## 設計判断

設計判断・経緯の記録は同ディレクトリの `design-notes.txt` を参照（非注入サイドカー）。

## プロジェクト上書き

- 上書き可否: 一律適用
- 理由: 単一HTML成果物の品質観点（被覆・図解・自己完結）はプロジェクトに依存しない普遍的な性質であり、緩和口を設けると合格基準が形骸化するため受け口を設けない
