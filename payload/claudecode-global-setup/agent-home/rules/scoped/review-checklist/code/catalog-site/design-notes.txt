# catalog-site 設計判断・経緯（非注入サイドカー）

## 設計判断（ADR）

### rule.md（観点表本体）

- 必要性: link-keeper と ai-slide-library という同型の蓄積型カタログサイトが並立し、品質がサイトごとにブレていた（実測: ai-slide-library はダークモードなし・絞り込みの畳みなし・全件一括描画）。共通の照合表がないとレビューが記憶頼みになり、同型サイトの新設のたびに品質が振れる。
- 代替案を採用しなかった理由: 各リポジトリの project rules に置く案は、同型サイト新設のたびに観点の複製と乖離が生じる。code/ui への追記案は、全 Web UI 共通基準（アイコン）と製品類型固有基準で適用ゲートが異なり、1 ファイルの肥大化を招く。
- 保守責任者: 人手（ユーザー）。観点の追加はユーザー指示または実事故を根拠に行う。
- 廃棄条件: 蓄積型カタログサイトの運用を全て終了した時。

## 経緯
