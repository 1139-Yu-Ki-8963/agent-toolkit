# 蓄積型カタログサイト基準（CATALOG-SITE-STANDARDS）

アイテムを継続蓄積し、検索と多軸絞り込みで探せる静的カタログサイトの実装に適用する合格基準。作成時に守るべき規約であると同時に、レビュー時の照合観点表そのものである。レビューでは code-reviewer が本ファイルを Read して照合する（フォルダ `review-checklist/code/` 配下 = code-reviewer 担当）。

観点を増やす場合は「意味語キー + 検査内容」の形式で該当カテゴリの表に追記する（キーは連番禁止。内容を要約した意味語で付ける）。既存観点の削除は禁止（検査の弱体化になるため。表現の具体化は可）。

## 適用ゲート

照合の前に、対象が「蓄積型カタログサイト」かを判定する。

- 該当する例: リンク集カタログ・スライドカタログ等、「アイテムの継続蓄積 + 検索窓 + 分類軸の絞り込み + カード一覧」を備えるカタログ画面（index.html 等）と、その生成スクリプト
- 該当しない例: カタログに登録される個別アイテム本体（スライドHTML等は document/html-output が担当）・単発のレポートHTML・アプリケーションの業務画面
- 該当しない場合は全観点を「対象外」とし、即 PASS として報告する

## 前提

- 規模前提: アイテムは数年で 1,000 件以上に成長する。規模に関わる観点はこの前提で照合する
- 自己完結性は要求しない。静的ホスティングでの配信を前提とし、CDN・外部フォントへの依存を許容する

## ヘッダー必須要素

| 観点キー | 検査内容 |
|---|---|
| ヘッダー-検索窓 | ヘッダー内に検索窓が常設され、ページ遷移なしで一覧が絞り込まれること |
| ヘッダー-ダークモード | 切り替えボタンがあること。初回訪問時は OS 設定（prefers-color-scheme）を初期値とし、選択を localStorage で永続化し、再読み込み時のちらつきを head 内の早期適用で防ぐこと |
| ヘッダー-ヒット件数 | 表示件数と全件数が常時表示され、変化がスクリーンリーダーに通知されること（aria-live） |

## 検索の質

| 観点キー | 検査内容 |
|---|---|
| 検索-正規化 | 大文字小文字・全角半角の揺れを吸収してマッチすること |
| 検索-対象明示 | 検索対象（タイトル・タグ・説明等）をプレースホルダー等で読み手に示すこと |
| 検索-デバウンス | 入力のたびに全件再計算・再描画せず、入力の確定を待ってから描画すること |

## 絞り込み

| 観点キー | 検査内容 |
|---|---|
| 絞り込み-畳み | 絞り込みパネルは開閉可能（aria-expanded 付き）で、モバイル幅では初期状態で折り畳まれていること |
| 絞り込み-選択カウンタ | 選択中の条件数が数値で表示されること |
| 絞り込み-一括クリア | 選択中の全条件を1操作でクリアできること |
| 絞り込み-0件disabled | 該当0件になる選択肢は disabled 表示とし、クリック不可とすること |
| 絞り込み-複合規則明示 | 複数軸選択時の規則（軸間AND・軸内OR等）が定義され、実装と一致すること |
| 絞り込み-件数交差定義 | 選択肢に添える件数の母数（全体か、他条件適用後か）を定義し、「N件と表示されるのに選ぶと0件」を起こさないこと |
| 絞り込み-URL状態 | 検索語・絞り込み状態が URL に反映され、共有可能であり、戻るボタンで復元されること |

## 一覧表示

| 観点キー | 検査内容 |
|---|---|
| 一覧-初期描画上限 | 初期描画するカード数に数値上限を定めること（目安30〜60件）。全件一括の DOM 構築を禁止する（1,000 件超で破綻するため） |
| 一覧-明示追加読み込み | 上限超過分は「もっと読み込む」ボタン等の明示操作で段階表示すること。スクロール連動の自動読み込みは禁止する |
| 一覧-ソート明示 | 現在の並び順が UI 上で明示されていること |
| 一覧-空結果案内 | 0件時に条件緩和の案内を表示すること |

## カード構成

| 観点キー | 検査内容 |
|---|---|
| カード-情報統一 | 全カードが同一の情報構成（タイトル・概要・タグ・日付等）を持つこと |
| カード-タグ連動 | 分類タグが表示され、タグの選択で該当軸の絞り込みに連動すること |
| カード-鮮度バッジ | 登録から閾値を超えたアイテムに警告バッジを表示すること。判定の基準時刻（閲覧時）を実装に明記すること |
| カード-画像欠損フォールバック | サムネイル画像が404の場合に壊れた画像を出さず、代替表示に落ちること |
| カード-画像遅延と寸法 | 画像に loading="lazy" と width/height を指定し、読み込みによるレイアウトのずれを防ぐこと |

## データ管理

| 観点キー | 検査内容 |
|---|---|
| データ-正本分離 | 表示用 HTML のデータ部は手編集禁止とし、正本ファイルからの再生成のみで更新すること |
| データ-ビルド時検証 | 生成時に必須フィールド欠落・日付形式不正・語彙外タグ・参照ファイル（サムネイル等）の欠損を検出して失敗させること |
| データ-語彙統制 | タグは管理された語彙一覧から選び、自由に増殖させないこと |
| データ-初期転送分離 | 一覧描画に使わない大容量フィールド（導入プロンプト全文等）は、一覧の初期読み込みデータから分離すること |

## セキュリティ・公開

| 観点キー | 検査内容 |
|---|---|
| XSS-エスケープ | カタログデータを HTML へ挿入する全箇所でエスケープすること |
| 外部リンク-noopener | target="_blank" のリンク全てに rel="noopener" を付けること |
| 公開-機密混入禁止 | データに特定環境の絶対パス・顧客情報を含めないこと |

## メタ・フォールバック

| 観点キー | 検査内容 |
|---|---|
| メタ-OGP | title・description・OGP・favicon を備え、URL 共有時に体裁が保たれること |
| メタ-ビューポート | viewport 指定があること |
| メタ-noscript | JavaScript 無効時に案内を表示すること |

## モバイル表示

| 観点キー | 検査内容 |
|---|---|
| モバイル-横スクロール禁止 | 狭幅（375px相当）でページ本体が横スクロールしないこと。幅を超えうる要素（タグ列・コード片等）は要素内スクロールに閉じること |
| モバイル-タップ領域 | チップ・ボタン・カード内操作のタップ領域が 44×44px 相当以上であること |
| モバイル-入力拡大防止 | 検索窓の input の font-size を 16px 以上とし、フォーカス時の画面自動拡大を防ぐこと |
| モバイル-ホバー非依存 | ホバーでのみ到達できる情報・操作を作らないこと。タップで同等の操作ができること |
| モバイル-1カラム成立 | 狭幅でカードが1カラムに畳まれ、情報の欠落・要素の重なりがないこと |
| レスポンシブ-グリッド | カードグリッドが画面幅に応じて列数を自動調整すること（auto-fill 等） |
| レスポンシブ-検索窓 | 狭幅でも検索窓が操作可能な幅を維持すること |

## アクセシビリティ

| 観点キー | 検査内容 |
|---|---|
| a11y-キーボード | 検索窓・絞り込み・カードリンクが Tab キーで到達可能であること |
| a11y-チップ状態 | 選択式チップの選択状態を aria-pressed 等で提示すること |
| a11y-コントラスト | ライト・ダーク両方の表示で WCAG AA（4.5:1）のコントラスト比を満たすこと |

## 機械強制

現時点では hook による機械強制なし。reviewing-against-rules 経由の code-reviewer 照合が防衛線となる。

## 違反検知時の手順

作成中・レビュー中に本基準への不適合を発見した場合:

1. 該当カテゴリ（ヘッダー / 検索 / 絞り込み / 一覧 / カード / データ管理 / セキュリティ / メタ / モバイル / アクセシビリティ）を特定する
2. 当該サイト内で修正する
3. 修正後、全観点を再照合してから完成として扱う

## 設計判断

設計判断・経緯の記録は同ディレクトリの `design-notes.txt` を参照（非注入サイドカー）。
