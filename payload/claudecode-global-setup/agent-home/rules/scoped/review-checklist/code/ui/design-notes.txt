# アイコン使用規約（ICON-USAGE）設計ノート（非注入サイドカー）
# rule.md から分離した保守者向け記録。規約の挙動には影響しない。

## 設計判断

**必要性**: 複数プロジェクト（既存プロジェクト A、既存プロジェクト B、既存プロジェクト C）で UI アイコンの統一が必要。Google Material Symbols Outlined は CDN 配信で依存パッケージ不要、アイコン数が豊富、全プロジェクトで共通利用できる。

**代替案を採用しなかった理由**:
- Lucide / Heroicons: npm パッケージ依存が増える。CDN 配信の手軽さに劣る
- SVG 直書き: アイコン数が増えるとコードが肥大化する
- 絵文字: デバイス間で表示が異なり、デザインの統一性が損なわれる

**保守責任者**: 人手（ユーザー）

**廃棄条件**: 全プロジェクトが別のアイコンシステムに統一された時
