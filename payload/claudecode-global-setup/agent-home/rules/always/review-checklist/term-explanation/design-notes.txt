# 略称・省略語の無断使用禁止（TERM-EXPLANATION）設計ノート（非注入サイドカー）
# rule.md から分離した保守者向け記録。規約の挙動には影響しない。

## 設計判断

### check-term-explanation.sh

**必要性**: 略称の無断使用は長時間セッションで頻発し、CLAUDE.md の文体規約や本 rule.md（常時注入）だけでは遵守率が落ちる（ユーザーが繰り返し指摘する実績あり）。最終応答の事後検査は Stop hook でしか実現できず、代表的な不透明略称がグロスなしで出現したら書き直しを強制する必要がある。ブロックリストの走査・グロス有無判定・自動解除カウンタという複数分岐があり、settings.json のインライン command には収まらない。

**代替案を採用しなかった理由**:
- CLAUDE.md / rule.md のみ（hook なし）: 常時注入しても長時間セッションで破られる実績がある。機械的な事後検査が要る
- PostToolUse: テキスト出力（ツール呼び出しを伴わない応答）を検査できない。最終応答全体を見るには Stop
- Bash ツール直叩き: Stop hook は harness 自動起動であり Bash ツール経由では発火経路がない

**保守責任者**: 人手（ユーザー）。ブロックリストの語追加・グロス判定の調整時に更新する。

**廃棄条件**: Claude Code 本体が応答内容の平易さ検査を組み込んだ時、または本規約が撤回された時。
