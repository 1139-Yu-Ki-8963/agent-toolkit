# 文章置き換え辞書規約（TEXT-DICTIONARY）設計ノート（非注入サイドカー）
# rule.md から分離した保守者向け記録。規約の挙動には影響しない。

## 設計判断

### prh.yml（サイドカー分離）

**必要性**: prh 辞書の正本が実行エンジンと同じ `~/agent-home/tools/linter/` に同居しており、「規約の正本は rules 配下に置く」というアーキテクチャ方針と矛盾していた。rules 配下に正本を移設しつつ、語彙の長大な定義を毎セッションの常時注入から外すため、`rule.md`（薄い索引）と `prh.yml`（非注入サイドカー）に分離した。`naming-values.txt` と同じ方式で、`.yml` 拡張子のファイルは rules 配下でも常時注入対象に含まれない。

**代替案を採用しなかった理由**:
- `tools/linter/` に残置: 規約の正本が rules の外に出続け、アーキテクチャ方針と矛盾したままになる
- symlink での参照: 実体とリンク先が分離し、どちらが正本か混乱を招く
- プロジェクト側から `prh` の `imports` でグローバルを参照する構成: 依存方向が「プロジェクト → グローバル」と逆転し、グローバル hook 側での合成ロジックが不要になる代わりに各プロジェクトの `.textlintrc` に個別実装が必要になる

**保守責任者**: 人手（ユーザー）。語彙追加・変更時は writing-quality スキルの登録フローに従って `prh.yml` を更新する。

**廃棄条件**: textlint / prh による文章検査を廃止した時。
