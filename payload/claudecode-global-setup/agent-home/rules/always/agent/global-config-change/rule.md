# グローバル設定資産の変更運用規約（GLOBAL-CONFIG-CHANGE）

グローバル設定資産（ルール・サブエージェント・CLAUDE.md・hooks設定・グローバルスキル）の変更を、受け渡しフォルダの指示書経由で管理する運用規約。

## 規約の要点

1. **起票が基本**: グローバル設定資産の変更は、受け渡しフォルダ（実行側の Projects 直下にある宛先バンドルと同名のフォルダ）の指示書への起票を基本とする
2. **暫定対処の許容と記録義務**: 配備側（agent-home リポジトリ）への直接修正は暫定対処として許容する。ただし、当該項目の表題末尾に【暫定対処済み】を付け、修正内容を指示書本文に自己完結で記載して、正本が追いつける状態を保つことを必須とする
3. **正本は payload**: agent-home リポジトリ（payload）が正本である。payload 反映の確認をもって指示書の当該項目を削除する

## 背景

実行側セッションが運用を知らずに配備側を直接修正し、正本が追いつけない変更が生まれることを防ぐ。本規約は 2026-07-14 のユーザー指示に基づき制定された。

## 公開資産の固有値分離

公開リポジトリ（agent-toolkit）へ同期される資産（同期対象は agent-toolkit の `scripts/sync-manifest.json` の mappings が唯一の定義）の本文に、プロジェクト固有名を直書きすることを禁止する。

1. 例示・記録で固有名が必要な場合は `<project>` プレースホルダーで書く
2. 実値が必要な設定は payload 除外機構（`scripts/payload-artifacts.json` の names 登録、または `.local.*` サフィックス）配下の非公開ファイルに分離し、公開資産はパス参照のみとする
3. 同期時の内容検査（forbiddenContent）の検査文字列リストは、`~/agent-home/state/payload-forbidden-content.json`（リポジトリ外・同期対象外）で管理する

## 機械強制

現時点では hook による機械強制なし。本規約の遵守は行動規範として運用する。

## プロジェクト上書き

- 上書き可否: 一律適用
- 理由: グローバル設定資産の変更管理はプロジェクトに依存しない普遍的な運用規約であり、受け口を設けない
