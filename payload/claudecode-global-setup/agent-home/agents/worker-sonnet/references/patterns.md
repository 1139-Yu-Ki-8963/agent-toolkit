# 調査・修正パターン集

worker-sonnet が参照する定型パターン。

## 調査パターン

### 呼び出し元追跡
```bash
grep -rn "関数名" --include="*.ts" --include="*.tsx" .
```
→ 結果を「直接呼び出し / 間接参照 / テスト内」に分類して報告

### 影響範囲分析
1. 変更対象の export を特定
2. import している全ファイルを列挙
3. 各ファイルでの使用箇所を確認
4. 波及的に影響するファイルを再帰的に追跡

### 規約チェック
1. 対象ファイルを glob で収集
2. 各ファイルを Read して規約条件を確認
3. 違反箇所を「ファイル:行番号:違反内容」で一覧化

### 整合性確認
1. 設定ファイル A の定義一覧を取得
2. 設定ファイル B の参照一覧を取得
3. A にあって B にない / B にあって A にないものを報告

## 修正パターン

### 規約違反の是正
1. checker の報告を受け取る
2. 各違反箇所を Read で確認
3. 規約に合うように Edit で修正
4. 変更ファイル一覧を報告

### リファクタリング
1. 変更方針を確認（brain の指示）
2. 影響範囲を先に調査
3. 本体を変更
4. 参照元を全て更新
5. 変更一覧を報告

## QA パターン（旧 qa-analyzer / qa-reviewer の知識）

### PR 分析
1. `gh pr diff <N>` で差分取得
2. 変更ファイルの分類（新規 / 修正 / 削除）
3. 各変更の影響範囲を分析
4. テスト観点の洗い出し

### テストケースレビュー
1. テストケースの網羅性確認
2. 重複の検出・除去
3. 優先度の妥当性確認（P0/P1/P2）
4. 不足ケースの指摘
