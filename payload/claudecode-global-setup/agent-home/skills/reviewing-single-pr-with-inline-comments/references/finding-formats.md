# 指摘の記録フォーマット詳細

Phase 5 で作る指摘の記録形式と、Phase 8 で投稿するインライン・総括コメントの本文テンプレート。SKILL.md 本体の Phase 5 から参照される。

## A. 内部記録形式（指摘リスト）

各指摘を以下の YAML-like 形式で記録する。Phase 8 でインラインと総括の両方を組み立てるための一次データとなる。

```
- id: <ラベル例: D-CACHE-01>
  severity: critical | warning | suggestion
  title: <短い 1 行タイトル>
  file: <相対パス>
  line: <行番号>           # 特定できない場合は null
  side: RIGHT | LEFT        # 既定 RIGHT
  what_happens: |
    <ユーザー視点の現象を 2〜5 行。「いつ / どの操作で / どうなる」を含める>
  why: |
    <コード視点の原因。該当箇所 file:line を bullet で並べ、必要なら関連箇所一覧表を添える>
  fix: |
    <対応案。選択肢が複数なら「案 A / 案 B」で分け、それぞれ 3 ステップ以内で書く>
  inline_summary: |
    <該当行用の 1〜2 行要約。インライン本文に流用する>
  related_locations:        # 同一指摘が複数行に跨る場合のみ
    - <file:line>
    - <file:line>
```

## B. 総括コメント用「重大な問題」セクション

`severity: critical` の各指摘は次の固定構造で書く。

```markdown
### 重大な問題 N: <title>

#### 何が起きるか（ユーザー視点）

<what_happens の本文>

#### なぜ起きるのか（コード視点）

<why の本文。必要なら現コードを ```python ... ``` で引用する>

<related_locations がある場合は次の表を添える>

| トリガー（コード位置） | 必要な対応 | 現状 |
|---|---|---|
| ... | ... | OK / **欠落** |

#### 対応案

<fix の本文>

該当行: [インラインコメント](<インライン投稿の HTML URL>)
```

## C. 総括コメント用「警告」セクション

警告は 1 件 1 段落で要約し、末尾にインラインへのリンクを添える。

```markdown
- **[<id>] <title>**（<file:line>）<inline_summary 1 行> — [インライン](<URL>)
```

## D. インラインコメント本文（コード行直結）

```markdown
**[<id>] <title>**

<inline_summary 本文 1〜2 行>

修正案:
```<lang>
<提案コード 1〜5 行>
```

詳細: 総括コメント「重大な問題 N: <title>」または「警告: <title>」を参照
```

## 重大な問題の必須要件（記入抜けの防止）

`severity: critical` を 1 件でも作る場合は次の 3 つすべてを埋める。1 つでも欠ければ severity を warning に下げる（または file:line を特定し直す）。

- `what_happens` … 画面/API 利用者から見た現象が書かれている
- `why` … 該当 file:line を最低 1 件示し、関連箇所が複数なら表で網羅する
- `fix` … コードを 1 行変える / PR 本文に 1 行書き足す等、実行可能な選択肢が示される

## 指摘フォーマットの模範パターン

重大な問題 2 件をそれぞれ「現象 → 原因 → 対応案」で展開し、関連箇所を表で俯瞰し、対応案を案 A / 案 B に分けるパターンが模範となる。総括コメントとインラインコメントを併用し、総括の各指摘から該当インラインへリンクを貼る。
