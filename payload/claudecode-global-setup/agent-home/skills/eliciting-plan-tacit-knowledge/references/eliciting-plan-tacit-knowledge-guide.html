計画暗黙知の抽出ガイド
eliciting-plan-tacit-knowledge

計画書をコンテキストゼロの弱モデルに初見読解させ、理解のズレから省略された前提（暗黙知）を抽出して計画へ明文化するスキル。

§1概要

計画の書き手は自分のコンテキストを前提に記述を省略する。書き手自身は読み直しても脳内で自動補完してしまうため、省略に気づけない。コンテキストゼロの弱い読み手に計画だけを読ませると、行間を埋められず省略箇所で理解が壊れる。その壊れ方（誤読・推測・判断不能）が暗黙知の地図になる。

この検出は 3 つの成立条件がそろって初めて機能する。1 つでも欠くと検出力が落ちる。

成立条件実現手段破るとどうなるか
読み手がコンテキストゼロ新規サブエージェント委任（プロンプトに計画文のみ埋め込み）メインセッションが初見読者を演じても意図を知っているため検出不能
読み手が弱いモデルplan-comprehension-prober（model: haiku 固定）強いモデルは行間を補完してしまい、暗黙知への依存が顕在化しない
「わからない」を言わせる形式推測・判断不能の明示列挙を強制する 5 項目出力曖昧な要約だけが返り、ズレの位置を特定できない

§2出力

このスキルを実行すると、次の 3 つが得られる。

ギャップ判定レポート
プローブの理解と本来の意図のズレを誤読・欠落・曖昧・偶然の的中の 4 分類で列挙したもの。欠落が暗黙知の本体。
改訂済み計画
抽出された暗黙知が「前提・制約（暗黙知の明文化）」節として追記され、誤読・曖昧の該当記述が書き換えられた計画。
再発暗黙知の候補報告
複数の計画で繰り返し抽出される同種の暗黙知。rules・テンプレート化の候補としてユーザーへ報告される。

§3用語

暗黙知
計画の書き手が無自覚に省略した前提・制約。
理解プローブ
コンテキストゼロの弱モデルに計画だけを読ませ理解を言語化させる検査。
ギャップ
プローブの理解と本来の意図のズレ。
収束
再プローブで新規の欠落・曖昧が 0 件になった状態。

§4動作フロー

Phase 1 対象と意図ソースの確定
  ├─ 対象計画（ファイル or 会話中テキスト）を特定
  └─ 意図の正解ソース（元指示・issue・設計書・会話決定事項）を Read

Phase 2 理解プローブ
  └─ plan-comprehension-prober へ委任
       prompt = [CHECKLIST-EXEMPT] + 計画全文ベタ書き + 5項目出力形式指示
       ※ 意図・背景は渡さない

Phase 3 ギャップ判定（呼び出し元が「決める」）
  └─ プローブ報告 × 意図ソースを突合し 誤読/欠落/曖昧/偶然の的中 に分類
       → 各ギャップの修正文をこの Phase で起草・確定

Phase 4 計画への再統合（worker-sonnet が「反映する」）
  └─ 確定済み修正文をベタ書きで渡し、計画ファイルへ反映

Phase 5 収束確認
  └─ 改訂後の計画全文で Phase 2 を再実行（新規 prober 呼び出し）
       新規の欠落+曖昧が0件 → 収束
       残あり → Phase 3〜5 を繰り返す（最大2周）

§5使い方

/eliciting-plan-tacit-knowledge で手動起動する。「暗黙知を炙り出して」「計画の前提を明文化して」と発話しても起動する。推奨実行位置は brain の計画立案後・ユーザー承認前。

制約: prober には意図・背景・補足を渡さない。混ぜた分だけ検出力が落ちる。

§6ギャップの4分類

分類定義対応
誤読書いてあるのに別の意味に取られた誤解を誘導した記述を書き換える
欠落書かれておらず、推測で埋められた・埋められなかった前提明文化して追記する（暗黙知の本体）
曖昧複数解釈が可能と申告された一意になるよう書き換える
偶然の的中推測したが偶然意図と一致していたそれも明文化する（次の読み手が同じ推測をする保証はない）

§7サブエージェント委任仕様

呼び出し箇所subagent_typeprompt 骨格期待返却値
Phase 2 / Phase 5 plan-comprehension-prober [CHECKLIST-EXEMPT] + 計画全文ベタ書き + 5項目出力形式の遵守指示。意図・背景は渡さない ①目的の言い換え ②手順の再説明 ③推測で補った前提 ④判断できなかった点 ⑤最初の一手
Phase 4 worker-sonnet 対象ファイルパス + 確定済み修正文（変更前後の対）+ スコープ外変更の禁止 + 完了条件（grep で確認可能な基準） 反映結果と変更箇所の報告

§8ループと収束

要素内容
反復条件Phase 5 で新規の欠落・曖昧が検出されたら Phase 3 → 4 → 5 を再実行する
上限回数2 周
停止条件収束停止（新規の欠落 + 曖昧 0 件）/ リソース上限（2 周到達）/ 発散検知（同一ギャップが 2 回連続で解消されない）

検証役（plan-comprehension-prober）と生成役（呼び出し元セッション + worker-sonnet）は分離されている。

§9予想を裏切る挙動

  • 1 周目で「全部合っている」報告が返った場合、成功と即断せず prompt への補足情報混入を疑う（意図まで書いてあれば当然一致する）
  • 「偶然当たった推測」も修正対象になる。合っていたから放置、は次の読み手で再発する
  • 再プローブは毎回新規呼び出し。同一 prober へ「修正版です、再確認して」と続けると前回文脈で補完され検出力を失う

§10関連スキル・規約

  • 字句レベルの曖昧語検出（廃止済み。曖昧表現の検出は document-reviewer の照合観点に統合）。本スキルは計画全体の暗黙の前提を対象とし、対象範囲が異なる
  • reviewing-against-rules — 成果物のルール照合レビュー。本スキルは計画文の理解可能性を対象とし合否は宣言しない
  • brain — 計画立案。本スキルは brain が立てた計画の品質確認を担う
  • subagent-selection 規約（~/agent-home/rules/always/agent/subagent-selection/rule.md）— 4 分類体系・判断と反映の分離の正本
