§1概要

このスキルは、textlint の置き換え辞書（prh.yml）に新しい語彙を登録するワークフロー専用。既存エントリでの置換修正・textlint エラーの読み方は ~/.claude/rules/always/review-checklist/text-dictionary/rule.md の違反時手順が完結して担う（本スキルへの委任なし）。

§2出力

このスキルを実行すると次が得られる。

更新済みの prh.yml
新規語彙の expected（推奨置き換え）・patterns（カタカナ形・英語形）を追加した辞書ファイル
既存使用箇所への置換適用
grep で発見した既存文書内の該当語を、誤検知注意に該当しない箇所について推奨語へ置換した状態

§3用語

prh辞書
textlintのprhプラグインが読む置き換えルール集。expected（推奨表記）とpatterns（検出対象の正規表現）の対で構成される
グローバル辞書 / プロジェクト辞書
グローバル辞書は全プロジェクト共通、プロジェクト辞書は<repo>/.claude/rules/always/review-checklist/text-dictionary/prh.ymlでグローバルに追加合成される（追加のみ可）

§4使い方

Step内容
Step 1登録先の判断。全プロジェクト共通の語彙はグローバル辞書、特定プロジェクト固有の語彙はプロジェクト辞書（受け口）に登録する
Step 2重複チェック。prh.yml を Read し、expected: 値で衝突がないか確認する
Step 3推奨置き換えと誤検知注意の決定。推奨置き換えは2〜4候補をスラッシュ区切りで併記し、誤検知注意は確立した訳語・固有API名等の対象外条件を明記する
Step 4辞書への追加。prh.yml の rules: 末尾に expected・推奨コメント・誤検知注意コメント・patterns を含む YAML ブロックを追記する
Step 5既存使用箇所の調査と置換。grep で既存文書内のヒットを確認し、誤検知注意に該当しない箇所を推奨置き換えで Edit する
Step 6検証とコミット。再度 grep を実行し残存が辞書自身のエントリ行・誤検知注意対象のみであることを確認し、grouping-commits で同一コミットにまとめる

§5動作フロー

prh.yml に該当エントリなし
  → Step1 登録先判断（グローバル/プロジェクト）
  → Step2 重複チェック
  → Step3 推奨置き換え・誤検知注意の決定
  → Step4 prh.yml へ追加
  → Step5 既存使用箇所を grep → 置換
  → Step6 grep再実行で検証 → grouping-commits でコミット

§6登録先の判断

種別パス用途
グローバル辞書~/.claude/rules/always/review-checklist/text-dictionary/prh.yml全プロジェクト共通の語彙を登録する正本
プロジェクト辞書<repo>/.claude/rules/always/review-checklist/text-dictionary/prh.yml特定プロジェクト固有の語彙を登録する受け口。グローバルと合成されてlintに使われる（追加のみ可・グローバル語彙の無効化は不可）

§7検証内容

検証項目: Step6の再grepで、残存が辞書自身のエントリ行または誤検知注意対象のみであることを確認する。

§8対象外パターン

技術略語（API・JSON・SDK 等）・ツール名（GitHub・Slack 等）・確立した訳語のない技術用語（コンテナ・リポジトリ等）は登録しない。

§9関連スキル・規約

text-dictionary/rule.md
辞書規約・違反時の自己完結修正手順の正本
grouping-commits
辞書追加と既存置換を同一コミットにまとめる際に使うスキル

§10予想を裏切る挙動

prh.yml の YAML インデントが崩れると linter が全体をパースエラーにする。追記後は必ずインデントを目視確認する。
