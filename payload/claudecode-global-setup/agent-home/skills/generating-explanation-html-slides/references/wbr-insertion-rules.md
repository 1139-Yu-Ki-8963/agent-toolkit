# 文節改行基準（`<wbr>` 挿入ルール）

HTML スライドの見出し・帯テキスト等の短文要素に `<wbr>` を挿入し、日本語の文節単位で改行させるための基準。Step 3-2 でこのファイルを Read し、基準に従って `<wbr>` を挿入する。

## 対象要素

- 見出し（h1〜h3）、帯テキスト（`.panel-tag` `.ribbon` 等）、キャプション、表セルの短文
- 本文段落は対象外（通常の折り返しで十分）

## 挿入位置（文節境界）

以下の位置に `<wbr>` を挿入する。

- 格助詞・副助詞の直後（は・が・を・に・で・と・も・の・へ・から・まで・より・など・って・では・には・とは・ほど・だけ・しか・ばかり）
- 読点「、」句点「。」の直後
- 接続助詞の直後（ので・けど・けれど・ながら・ために・たら・れば・つつ）
- 連用中止形の切れ目（〜し、/ 〜して / 〜すると / 〜された + 次文節）
- 名詞 + 動詞の境界（「設計を行う」の「を」後ではなく「設計を」と「行う」の間）

### 境界ケースの判断原則

上記ルールで判断がつかない場合（複合動詞・慣用句・長い連体修飾句など）は「意味のまとまりを割らない」を優先する。機械検証（`check-wbr-quality.sh`）は禁則違反のみを検査し、挿入位置の妥当性は document-reviewer の目視判定に委ねる。

## 禁則（挿入しない位置）

- カタカナ連続語の途中（「インフラストラクチャ」を割らない）
- 漢字熟語の途中（「機械学習」を割らない）
- 固有名詞の途中（「Claude Code」を割らない）
- 数値 + 助数詞の間（「100件」「3回」を割らない）
- 2 文字以下の断片を生む分割（最小チャンク 3 文字）
- 括弧・引用符の内部で対の途中を割らない

## 免除条件

- 8 文字未満の見出し
- 日本語を含まない見出し（英語のみ等）

## 例

### 正しい挿入

```html
<h2>解説スライドの<wbr>文節改行を<wbr>機械検証する</h2>
```

### 禁則違反（NG）

```html
<!-- カタカナ連続の途中で分断 -->
<h2>インフラ<wbr>ストラクチャの設計</h2>

<!-- 2 文字以下の断片 -->
<h2>設<wbr>計を行う</h2>

<!-- 数値 + 助数詞の間 -->
<h2>100<wbr>件の処理を実行</h2>
```
