# 解説スライド観点レビュー表

generating-explanation-html-slides Phase 4 の合格判定の正本。生成したスライドHTMLを実描画したうえで、以下の全行を照合し PASS / NG を記録する。全行 PASS のみ保存・登録できる。

キーは連番禁止。内容を要約した意味語で付ける（番号からは情報を得られないため）。観点を追加する場合も同じ形式で追記する。

## 観点の2層構造と育成サイクル

観点は2層で管理する。本表は**全スライド共通の観点の正本**（第1層）、各スライドの `docs/10_解説スライド集/<スライドキー>/検査記録.md` が**タスク限定の観点**（第2層）を保持する。

育成サイクル（SKILL.md の Step 6-2 が実行する）:

1. 生成したスライドを本表で照合し、ユーザーレビューを受ける
2. 受けた指摘・修正ループで見つけた欠陥を原則として言語化する
3. 昇格判定: (a) 別トピックでも再発しうる (b) 意味語キー+検査内容+NG例の形式で書ける (c) 実描画またはファイル検査で検査可能 — 3つすべて満たせば本表へ追記する
4. 満たさない指摘（そのスライド固有の事実・制約）は検査記録.md に保持し、当該スライドの更新時の再検査項目とする

本表の既存観点の削除は禁止（検査の弱体化になるため）。表現の具体化・NG例の追加は可。本表の由来: 原型スライド（claude-cursor使い分け-リミット観点）の対話レビューで受けた指摘を、このサイクルの手順2〜3で昇格させたものが初期13観点。

## 共通観点（全型に適用・実描画スクリーンショットで判定）

| 観点キー | 検査内容 | NG例 |
|---|---|---|
| 文字切れ-全数目視 | 全テキスト要素に切れ・重なり・はみ出しがないか（1要素ずつ確認） | 幅の狭い帯で「Cursor」の右端が overflow: hidden で欠ける |
| 自己完結-外部参照なし | CDN・Webフォント・外部画像・fetch の参照が1つもないか | Google Fonts の link タグが残っている |
| 文字コード-charset明記 | 先頭に `<meta charset="utf-8">` があるか | charset なしでローカル配信すると日本語が文字化けする |
| 配色-ビジネス基調 | 白面+ネイビー系が基調で、彩度の高い色が識別・警告・回復に限定されているか | ダークモード配色・装飾目的の多色使い |
| 比率-スライド16対9 | 16:9 比率で 1280px 幅表示時にレイアウトが崩れないか | 縦に溢れてスクロールが発生する・要素が枠外に出る |
| 内部名称-混入なし | スライド内に consult- で始まる内部スキル名・内部パスが無いか | ヘッダに consult-slide 等の内部スキル名や内部ファイルパスがそのまま表示されている |
| 寸法-固定1280x720 | `.slide` の width/height が 1280px/720px 固定であること | `min()` や `aspect-ratio` による可変寸法 |
| 単位-固定px限定 | `.slide` 内に `vw`/`vh`/`clamp()` が存在しないこと（検証: `grep -nE 'vw\|vh\|clamp\(' file`） | `.slide` 内に `clamp(9px, 1vw, 11px)` が残っている |
| フッタ-区切り線 | メタデータフッター（`.slide-meta`）が区切り線+下端余白として存在し、内容が空であること | フッター内にテキスト（日付・作成者名・ページ番号等）が残っている |
| 具体性-実値明示 | 各カード・帯・パネルの説明に検証可能な具体（固有名・数値・形式名・実例）が最低 1 つあるか。具体は実在の仕組み・実測・実装から取ったものに限り、具体性を出す目的での値の創作を禁止する。実在性はファイル単体では照合できないため、作成時に出所（調査報告・規約本文・実測）を検査記録.md へ残すことで担保する | 「弱いモデルに読ませて検証する」とだけ書き、モデル名（Claude Haiku）・報告形式（5 項目）等の実体が 1 つもない |
| 効果数値-未検証記載禁止 | 効果・工数・時間の数値（短縮時間・削減率・見積もり等）が、検証記録または実在の出所なしに記載されていないか。裏付けのない効果数値は「例：」付記や緩和表現への書き換えではなく削除する | 検証していないのに「60 分かかる作業が 5 分に短縮」と記載する |
| wbr-文節改行 | 見出し・帯テキストの `<wbr>` が wbr-insertion-rules.md の基準に準拠しているか。check-wbr-quality.sh が PASS であること | 助詞の前で分割されている・カタカナ語の途中で分割されている |
| 配色トークン-hex直書き | :root のトークン定義ブロック以外に hex カラーの直書きが無いか（:root 外の #RGB/#RRGGBB 出現 0 件を grep で機械確認） | `.step-chip { color: #fff; }` のような部品側への白文字直書き |

## 図解スライド固有の観点（構造・意味の検査）

| 観点キー | 検査内容 | NG例 |
|---|---|---|
| 因果-構造表現 | 変化の引き金（上限到達・締切・承認等）がイベント線・バッジ等の構造部品で描かれているか | 2つのカードを矢印でつなぎ、引き金は矢印上の小さなラベルだけ |
| 識別子-実物想起 | 実在ツール・製品が実物ロゴを想起させる形+色+名称で描かれているか | 六角形・角丸四角など発明した抽象図形にツール名を添えただけ |
| 状態-極端表現 | 「尽きた・満杯・ゼロ」等の状態がメーター100%+警告色で極端に描かれているか | 「使い切った」の説明なのにゲージが72%で止まっている |
| 時間-軸割当 | 時間経過を含む説明で横軸が時間になっているか | 週次サイクルの説明なのに横軸がツールの並置になっている |
| 時間スケール-図分離 | 異なる時間スケール（時間単位/週単位等）がパネル分割されているか | 5時間の話と1週間の話が1本の軸に混在している |
| 例示数値-検算整合 | 図中の具体値が説明対象の仕組みと矛盾しないか（検算済みか） | 開始から5時間でリセットされる枠なのに、5時間使い切ってから切り替える時系列 |
| 読み順-番号対応 | 図中のステップ番号チップと本文要約の番号・色が一致しているか | 図に番号がない、または本文と番号の意味がずれている |
| 軸範囲-業務文脈 | 時間軸の範囲が業務文脈に合っているか（営業日のみ・業務時間帯等） | ビジネス資料の週次説明に土日が含まれている |
| カード空白バランス | 横並びカード（stages-grid / compare-row）で、コンテンツが少ないカードの内部に隣接カードの高さに引きずられた目立つ空白帯がないか。評価行がカード下端に揃っている場合、その上の空白は許容する | 3枚の段階カードのうち1枚だけ内容が多く、他2枚の下半分が空白のままになっている |

## 論証テーブル固有の観点

| 観点キー | 検査内容 | NG例 |
|---|---|---|
| 列役割-分離 | 「課題」列に対策、「現状」列に意見が混入していないか | 課題列に「〜を導入すべき」と対策を記載 |
| 打ち手-主体明記 | すべての打ち手に実行主体（誰がやるか）が入っているか | 「マニュアル化を推進」だけで主体なし |
| リード帯-結論自足 | リード帯+ヘッドメッセージだけ読めば表を見なくても結論が分かるか | リード帯が「ナレッジ管理について」というラベル型 |
| 行数-上限遵守 | 5行以内か。超える場合は分割案を提示しているか | 8行詰め込みで読めない |
| 打ち手-具体性 | 「〜の推進」「〜の強化」で終わる曖昧な打ち手がないか | 「DX推進の強化」で具体アクションなし |
| 現状-事実限定 | 現状列が事実と数字だけか（意見・評価が混入していないか） | 「対応が不十分である」という評価文 |
| 出典-明示 | データの出所が出典行に記載されているか | 数字があるが出典なし |
