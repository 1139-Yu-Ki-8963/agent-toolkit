# 定期ヘルスチェック（10 分ループ）

orchestrating-dev-flow の実行中に ScheduleWakeup で 10 分ごとにループを回し、フローの健全性を自動チェックする。

## 開始・停止

- **開始**: Phase 2（worktree 作成完了時）に ScheduleWakeup(600s) で開始
- **停止**: Phase 10（worktree 削除時）にループを停止（ScheduleWakeup を呼ばない）
- Phase 1 は調査のみのためループ不要

## チェック項目

### 毎回実行（10 分ごと）

1. **サブエージェント完了検知**
   - バックグラウンドで起動した Agent の完了通知を確認する
   - 完了していれば結果を処理し、次の Step に進む

2. **ユーザー指示の見落とし確認**
   - 未応答のユーザー発話がないか確認する
   - 見落としがあれば即対応する

3. **Phase 完了条件の自己診断**
   - 現在の Step の完了条件が満たされているか照合する
   - 満たされているのに次 Step に進んでいない場合は進行する

### 2 回に 1 回実行（20 分ごと）

4. **ルート昇格チェック**（Phase 6 のみ）
   - quick / refactor ルートで classify 条件の違反がないか確認する
   - 違反があれば full ルートに昇格する

5. **TaskCreate 漏れ確認**
   - 現 Phase の全 Step が TaskCreate 済みか確認する
   - 漏れがあれば追加登録する

6. **ステータスライン進捗ファイルの整合確認**
   - 最後の update-flow-status.sh 実行と実際の進捗が一致するか確認する
   - 不一致があれば更新する

## ループの動作

```
問題なし → ScheduleWakeup(600s) で次回を予約。ユーザーへの報告なし
問題あり → 1 行で報告し修正アクションに入る。修正後に次回を予約
サブエージェント完了 → 結果処理を優先。処理完了後に次回を予約
```

## ScheduleWakeup のプロンプト

```
実装フロー定期ヘルスチェック（10 分ループ）:
1. サブエージェント完了検知 → 完了していれば結果処理
2. ユーザー指示の見落とし確認
3. Phase 完了条件の自己診断
4. ルート昇格チェック（Phase 6 のみ、2 回に 1 回）
5. TaskCreate 漏れ確認（2 回に 1 回）
6. ステータスライン進捗ファイル整合確認（2 回に 1 回）
問題なし → 次の 10 分ループを予約。問題あり → 報告して修正。
```
