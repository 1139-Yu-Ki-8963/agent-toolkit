# グループ化ルール

### テスト関連
- **対象ファイル**: `__tests__/`, `.test.ts`, `.test.tsx`, `.spec.ts`
- **プレフィックス**: `【テスト】`
- **メッセージパターン**:
  - 新規テスト → `【テスト】[テスト対象]のテストを追加`
  - テスト修正 → `【テスト】[テスト名]の[問題点]を修正`
  - テスト改善 → `【テスト】[改善内容]`

### ドキュメント
- **対象ファイル**: `.md`, `docs/`, `CLAUDE.md`, `README.md`
- **プレフィックス**: `【ドキュメント】`
- **メッセージパターン**:
  - 新規 → `【ドキュメント】[ドキュメント名]を追加`
  - 更新 → `【ドキュメント】[更新内容]`

### 設定ファイル
- **対象ファイル**: `.json`, `.config.js`, `tailwind.config.js`, `.claude/`
- **プレフィックス**: `【設定変更】`
- **メッセージパターン**:
  - `【設定変更】[設定項目]を更新`
  - `【設定変更】[設定内容]`

### UI/コンポーネント
- **対象ファイル**: `src/components/`, `src/app/`, `.tsx`
- **プレフィックス**:
  - 新機能 → `【機能追加】`
  - 既存修正 → `【リファクタ】`
  - バグ修正 → `【バグ修正】`
- **メッセージパターン**:
  - `【機能追加】[コンポーネント名]を追加`
  - `【リファクタ】[改善内容]`
  - `【バグ修正】[修正内容]`

### ビジネスロジック
- **対象ファイル**: `src/domain/`, `src/application/`, `src/infrastructure/`
- **プレフィックス**:
  - 新機能 → `【機能追加】`
  - リファクタリング → `【リファクタ】`
  - バグ修正 → `【バグ修正】`

### スタイル/デザイン
- **対象ファイル**: `src/ui/`, `tokens.ts`, スタイル関連
- **プレフィックス**: `【スタイル】` または `【機能追加】`
