カバレッジ自動改善

保守

テストカバレッジが目標未満のファイルを検出し、不足テストを自動生成して PR を作成する。過剰テスト・トートロジーの品質検証も行う。

発火タイミング 日次クラウド Routine。profile.md の閾値（vitest 60% / pytest 70% 等）を下回るファイルを対象にする。

ステップ

4
1 profile.md を Read
プロジェクトの技術スタック・パス・コマンド・閾値を取得する。
2 カバレッジ計測
profile 指定のコマンドでカバレッジを計測し、閾値未満のファイルを特定する。
3 テスト自動生成
不足テストを生成し、トートロジー・過剰テストでないか品質検証する。
4 PR 作成
生成したテストを worktree でコミットし PR を作成する。

フロー図

profile.md → カバレッジ計測 → 閾値判定 → テスト生成 → 品質検証 → PR

関連スキル・hook

managing-agent-configs parallel-dev-worktree grouping-commits

注意・例外

  • PR 影響度: なし（マージまで）。
