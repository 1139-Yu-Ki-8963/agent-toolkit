ドキュメント健全性チェック

保守

docs/ 配下のドキュメント資産の腐敗（リンク切れ・孤立ファイル・文章品質劣化）を定点検出し、修正可能なリンク切れは PR で自動修正する。

発火タイミング 週次クラウド Routine（木曜 10:03 JST）。

ステップ

5
1 内部リンク検証
docs/ 配下の .md / .html ファイルの内部リンクを抽出し、参照先の存在を確認する。
2 孤立ファイル検出
他ファイルからリンクされていないファイルを検出する。
3 禁止パターン検出
.md ファイルに対し LLM 空句・ダッシュ・二人称の禁止パターンを grep 検出する。
4 PR 作成またはログ記録
修正可能なリンク切れがあれば PR を作成・マージする。修正不可能な場合はログ記録のみ。
5 結果ログ出力
run_end を記録する。

フロー図

内部リンク検証 → 孤立ファイル検出 → 禁止パターン検出 → PR作成/ログ記録

関連スキル・hook

managing-agent-configs

注意・例外

  • PR 影響度: なし（マージまで）。コンテンツ変更なし・リンクパス修正のみ。
