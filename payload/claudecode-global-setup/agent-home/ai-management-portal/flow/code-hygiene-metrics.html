コード衛生メトリクス

保守

lint では検出できない構造的品質劣化（lint 抑制の増加・ファイル肥大化・循環依存）を定点観測し、ログのみに記録する。

発火タイミング 週次クラウド Routine（木曜 10:06 JST）。

ステップ

4
1 lint 抑制コメント集計
FE の biome-ignore、BE の type: ignore・# noqa の件数を grep で集計する。
2 ファイル肥大化検出
wc -l で 500 行超のファイルを一覧化する。
3 循環依存検出
npx dependency-cruiser で FE の循環依存を検出する。
4 結果ログ出力
計測値を JSONL ログに記録する。

フロー図

lint 抑制カウント → ファイル肥大化検出 → 循環依存検出 → JSONL ログ出力

注意・例外

  • PR 影響度: なし（ログのみ）。
