コミット整形

commit

Git の変更を自動分析してグループ化し、関連する変更ごとに単一目的のコミットを作成する。機密ファイル混入もチェックする。

発火タイミング コード変更後にコミットを作成したいとき、複数の変更をまとめて整理したいとき。コミットを行わない閲覧・編集のみのときはスキップ。

ステップ

4
1 変更分析 grouping-commits
変更ファイルを test/docs/chore/feat/fix/refactor/style に自動分類する。
2 グループ化
関連する変更を 1 つの目的にまとめる。
3 機密チェック reviewing-public-readiness
コミット前に機密ファイルの混入を確認する（PUBLISH-SAFETY と連携）。
4 段階コミット
グループごとに git add + commit する。メッセージは命名規約（rules: always/naming/commit-branch）に従う。

フロー図

変更分析 → 目的別グループ化 → 機密チェック → グループごとに add+commit

関連スキル・hook

grouping-commits reviewing-public-readiness

注意・例外

  • issue-<番号> ブランチは managing-github-issues（verify モード）経由で 1 コミット化する。
  • commit 時に PreToolUse hook が [NAMING] [GIT-AUTHOR-BLOCK] [PUBLISH-*] を注入する。
