seed 即反映

運用

ローカルの supabase/seeds/*.sql 更新後、worktree→commit→push→PR 作成→マージ→git pull→supabase db reset→アプリ再起動までを自動実行する（- 専用）。

発火タイミング 「seed を反映して」「seed を更新した」「マージまでして」と言われ、supabase/seeds/*.sql に未コミット変更があるとき。seed 以外のコードも変更されている場合は parallel-dev-worktree を使う。

ステップ

5
1 worktree 作成 seed-deploy
seed 変更を隔離する worktree を切る。
2 commit + push grouping-commits
seed ファイルをコミットして push する。
3 PR 作成 + マージ formatting-pr
PR を作成しマージまで進める。
4 DB リセット
git pull 後に supabase db reset で seed を反映する。
5 アプリ再起動
ローカル開発アプリを再起動して反映を確認する。

フロー図

worktree → commit → push → PR → マージ → git pull → supabase db reset → アプリ再起動

関連スキル・hook

seed-deploy parallel-dev-worktree grouping-commits formatting-pr

注意・例外

  • seed ファイルに変更がない、または DB リセットのみ求められたときはスキップ。
