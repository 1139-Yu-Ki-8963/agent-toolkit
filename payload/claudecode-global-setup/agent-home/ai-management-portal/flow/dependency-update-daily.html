パッケージ脆弱性パッチ

保守

npm audit / pip-audit で CVE 脆弱性を検出し、patch / minor の安全な更新を PR にまとめる。major 更新は手動レビュー待ちとして記録する。

発火タイミング 日次クラウド Routine。

ステップ

4
1 profile.md を Read
パッケージマネージャと lock ファイルのパスを取得する。
2 脆弱性スキャン
npm audit / pip-audit で CVE を検出する。
3 安全更新
patch / minor レベルの更新を適用し、テストを実行する。
4 PR 作成
更新内容を PR にまとめる。major は記録のみ。

フロー図

profile.md → 脆弱性スキャン → patch/minor 更新 → テスト → PR

関連スキル・hook

managing-agent-configs parallel-dev-worktree

注意・例外

  • PR 影響度: あり（E2E 必須）。major 更新は自動適用しない。
