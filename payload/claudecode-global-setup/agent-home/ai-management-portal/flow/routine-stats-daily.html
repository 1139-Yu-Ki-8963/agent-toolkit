ルーティン統計レポート

運用

全ルーティンの JSONL ログを集約し、日次 Issue でサマリ・失敗詳細・週次トレンドを報告する。唯一 Issue 作成が許可されたルーティン。

発火タイミング 日次クラウド Routine。全ルーティンの実行後に起動する。

ステップ

4
1 profile.md を Read
集約対象のログディレクトリを取得する。
2 JSONL ログ集約
全ルーティンのログを読み込み、成功/失敗/スキップを集計する。
3 トレンド分析
過去 7 日間のトレンドを算出する。
4 Issue 作成
サマリ・失敗詳細・トレンドを日次 Issue として起票する。

フロー図

profile.md → JSONL 集約 → トレンド分析 → 日次 Issue 起票

関連スキル・hook

managing-agent-configs managing-github-issues

注意・例外

  • PR 影響度: なし（Issue のみ）。唯一 Issue 作成が許可されたルーティン。
