NO-DELEGATION 強制

hook

ユーザーへの操作依頼を全面禁止する。対話必須コマンドは deny、依頼文は書き直しを強制し、ログインは token ベース代替へ誘導する。

発火タイミング 最終応答や PR/issue 本文に依頼文（「〜してください」等）、または対話ログイン（gh auth login 等）が含まれるとき。

ステップ

4
1 deny
permissions.deny が対話必須コマンドを実行前に拒否する。
2 PreToolUse 検出
check-no-delegation-pre-bash.sh が [NO-DELEGATION-BLOCK] で exit 2 する。
3 書き直し
依頼文を削除し、Claude 自身のツールで完遂する応答へ差し替える。
4 代替不能なら中止
[NO-DELEGATION-ABORT] フォーマットで事実報告する。

フロー図

deny → PreToolUse [NO-DELEGATION-BLOCK] → 依頼文削除 → token 代替で完遂 / [NO-DELEGATION-ABORT]

注意・例外

  • 本番接続は PROD-SKILL-READ により -deploy-production 必読を強制する。
  • 同 hook が 2 連続発火したら 3 回目の書き直しをせず事実報告する。
