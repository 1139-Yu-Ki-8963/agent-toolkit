issue 選択 → 実装着手（pick モード）

起点

GitHub issue 一覧を表形式で表示し、ユーザーが番号を選んだら flow:* ラベルで統合開発フロー（orchestrating-dev-flow）を起動する。issue から開発を始める入口。

発火タイミング 「イシューを見て」「issue 一覧を出して」「課題を選んで実装して」など、issue 選択から実装を始めたいとき。issue 番号が既に明示されている場合はスキップ。

ステップ

4
1 issue 一覧取得 managing-github-issues
gh issue list でオープン issue を取得し、番号・タイトル・ラベルを表形式で表示する。
2 番号選択
ユーザーが対象 issue の番号を選ぶ。
3 フロー判定
issue の flow:* ラベルから単機能・一括・メンテのいずれかを判定する。
4 実装フロー起動 orchestrating-dev-flow
判定結果に応じて orchestrating-dev-flow を起動し、統合開発フローへ接続する。

フロー図

gh issue list → [表示] → 番号選択 → flow:* ラベル判定 → orchestrating-dev-flow 起動

関連スキル・hook

managing-github-issues orchestrating-dev-flow parallel-dev-worktree

注意・例外

  • issue 番号が既に分かっている、またはコードを直接修正する指示があるときは起動しない。
