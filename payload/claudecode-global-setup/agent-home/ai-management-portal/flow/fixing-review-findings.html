レビュー指摘の自動修正

fix

レビューで挙がった警告・critical を worktree 内で自動修正し、lint/test 再実行・再レビューを経て自動マージまで進める。

発火タイミング reviewing-prs / pr-review-workflow が指摘を出した後、または orchestrating-dev-flow の Phase 10 として起動されたとき。

ステップ

5
1 指摘取り込み module-fixing-review-findings
レビュー結果の警告・critical を一覧化する。
2 自動修正
worktree 内で該当箇所を Edit で修正する。
3 再検証
lint/test を再実行し、修正の正しさを確認する。
4 push + 再レビュー
PR ブランチへ push し、再レビューを依頼する。
5 自動マージ
残存指摘が解消したら自動マージする。

フロー図

指摘取込 → 自動修正 → lint/test 再実行 → push → 再レビュー → 自動マージ

関連スキル・hook

module-fixing-review-findings reviewing-single-pr-with-inline-comments

注意・例外

  • 修正不能な critical が残る場合はユーザーに事実報告して中断する。
