worktree 開発フロー

実装

origin から git worktree を切って隔離環境で実装し、PR 作成後に worktree を畳む。typo 修正・1 行変更も含め規模を問わず適用する。

発火タイミング 「実装」「修正」「直して」「対応」「issue 解決」と指示され、worktree が未作成のとき。読み取りのみの調査や、ユーザーが「main で直接」と明示した場合はスキップ。

ステップ

7
1 worktree 作成 parallel-dev-worktree
origin/main から新しいブランチの worktree を切る。命名は命名規約（rules: always/naming/commit-branch）に従う。
2 着手前チェック
対象範囲とコンフリクト可能性を確認する。
3 実装
worktree 内でコードを編集する。最小差分で根本から直す。
4 テスト
ユニット・E2E を実行し、動作を実証する。
5 コミット整形 grouping-commits
grouping-commits で目的別コミットを作成する。
6 PR 作成 module-formatting-pr
formatting-pr で PR 本文を整形し gh pr create する。
7 worktree 削除
PR 作成後に git worktree remove で畳む。

フロー図

worktree作成 → 着手前チェック → 実装 → テスト → grouping-commits → formatting-pr → review → worktree remove

関連スキル・hook

parallel-dev-worktree grouping-commits module-formatting-pr

注意・例外

  • ~/.claude/* ~/agent-home/* と git 管理外パスは WORKTREE 強制の例外パス。
  • 既存ブランチ継続中の追加コミットには適用しない。
