DB 日次バックアップ

DB

本番 DB のダンプを取得し、profile で指定された保存先（orphan branch / クラウドストレージ）に配置する。

発火タイミング 日次クラウド Routine。

ステップ

4
1 profile.md を Read
DB 接続情報と保存先を取得する。
2 ダンプ取得
pg_dump 等でバックアップを取得する。
3 保存先配置
orphan branch / クラウドストレージに配置する。
4 結果ログ出力
バックアップサイズと所要時間を JSONL ログに記録する。

フロー図

profile.md → pg_dump → 保存先配置 → JSONL ログ出力

関連スキル・hook

managing-agent-configs supabase

注意・例外

  • PR 影響度: なし（ログのみ）。
