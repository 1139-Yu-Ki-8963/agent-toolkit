PR 本文整形

PR

把握済みの実装内容を .github/pull_request_template.md に厳密準拠した PR 本文へ整形し、gh pr create で登録する。formatting-issue の PR 版。

発火タイミング 「これを PR にして」「PR 本文を書いて」など、実装完了済みで次の操作が gh pr create のみのとき。内容を対話的に決めたい場合や複数 PR 一括処理はスキップ。

ステップ

4
1 テンプレート読込 module-formatting-pr
.github/pull_request_template.md の見出し構造を読む。
2 本文組み立て
変更内容をテンプレートの各セクションへ埋める。文章は text-dictionary/rule.md の置き換え辞書に従う。
3 セルフチェック
先送り表現・依頼文・textlint 違反がないか確認する。
4 PR 作成
gh pr create で登録する。

フロー図

テンプレ読込 → 本文組み立て → セルフチェック（先送り/依頼/textlint）→ gh pr create

関連スキル・hook

module-formatting-pr

注意・例外

  • [TEXTLINT-BLOCK] [NO-DEFERRAL-BLOCK] 注入時は本文を再整形してから再実行する。
  • 「未実施・残課題」は「- なし」のみ許容（先送り全面禁止）。
