linter 自己進化

保守

実コードの傾向を分析し、lint ルールを自動拡張する。既存コードが通過する安全なルールのみ追加する。

発火タイミング 日次クラウド Routine。

ステップ

4
1 profile.md を Read
lint 設定ファイルのパスと対象ディレクトリを取得する。
2 コード傾向分析
実コードのパターンを分析し、追加候補ルールを特定する。
3 既存コード検証
候補ルールが既存コードを破壊しないか検証する。
4 PR 作成
安全なルールのみ lint 設定に追加し PR を作成する。

フロー図

profile.md → コード傾向分析 → ルール候補 → 既存コード検証 → PR

関連スキル・hook

managing-agent-configs

注意・例外

  • PR 影響度: なし（マージまで）。既存コードが通過するルールのみ追加する。
