仕様同期

同期

直近 24 時間のコード変更を検出し、対応する設計書との乖離を特定して同期 PR を作成する。

発火タイミング 日次クラウド Routine。

ステップ

4
1 profile.md を Read
コードと設計書の対応マッピングを取得する。
2 変更検出
直近 24 時間の git diff からコード変更を特定する。
3 乖離分析
変更箇所に対応する設計書との整合を検証する。
4 同期 PR 作成
乖離がある設計書を更新し PR を作成する。

フロー図

profile.md → git diff(24h) → 設計書との突合 → 乖離検出 → 同期 PR

関連スキル・hook

managing-agent-configs parallel-dev-worktree

注意・例外

  • PR 影響度: なし（マージまで）。
