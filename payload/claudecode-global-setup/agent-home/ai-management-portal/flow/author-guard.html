AUTHOR 強制

hook

commit/push の author を y__u 名義 1 件のみに限定する。PreToolUse hook と husky pre-push が二重に block する。

発火タイミング commit 前に [GIT-AUTHOR-BLOCK]、push 前に [GIT-AUTHOR-PUSH-BLOCK] が注入されたとき。

ステップ

2
1 commit 前検査
[GIT-AUTHOR-BLOCK] 時は ~/.gitconfig の [user] を y__u に直してから再実行する。
2 push 前検査
[GIT-AUTHOR-PUSH-BLOCK] 時は rebase --reset-author で author を書き換えてから再 push する。

フロー図

commit → [GIT-AUTHOR-BLOCK] gitconfig 修正 / push → [GIT-AUTHOR-PUSH-BLOCK] rebase --reset-author → 再 push

注意・例外

  • 許可: name=y__u / email=y__u@users.noreply.github.com の 1 組のみ。
  • --no-verify による bypass、GIT_AUTHOR_* の export、-c user.email のワンショット指定は禁止。
