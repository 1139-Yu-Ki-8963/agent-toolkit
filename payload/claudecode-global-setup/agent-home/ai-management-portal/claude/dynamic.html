git で追跡する／しない

本リポジトリは .gitignore で動的層を一括除外する設計。新規の動的ファイルを生むツールを追加した場合は .gitignore に追記し、誤コミットを防ぐ。

  • 追跡する: CLAUDE.md・settings.json・statusline.py・README.md・agents/・skills/・plugins/・rules/・config/
  • 追跡しない: 上表の「動的データ・キャッシュ・マーカー」全般

設計ドキュメント（本ポータル）は ~/agent-home/ai-management-portal/ に統合済みのため、~/.claude/ では git 追跡の対象外となった。

cleanup の指針
  • 容量肥大化が気になった時点で cache/・paste-cache/・uploads/・shell-snapshots/ を実 rm で削除する（CLAUDE.md セキュリティ節）
  • 古い sessions/ / projects/ は要約だけ残して transcript を整理する
  • backups/ は世代管理（直近 N 世代のみ残す）
  • worktree 由来の残骸は git worktree remove <path> で畳む（mv _trash 退避は廃止）
