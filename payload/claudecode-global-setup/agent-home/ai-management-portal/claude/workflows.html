機械強制ワークフロー

hook が注入するタグと Claude の応答手順を一覧化する。実装本体は ~/agent-home/tools/hooks/*.sh および ~/.claude/rules/ 配下にある。

タグ一覧
タグ発火 hook意味Claude の応答
[NAMING]PreToolUse(Bash)ディレクトリ・ファイル名の規約注意ケバブケース・予約名を避けて命名する
[GIT-AUTHOR-BLOCK]PreToolUse(Bash)commit 前に author が白リスト外~/.gitconfig の identity を直して再 commit
[GIT-AUTHOR-PUSH-BLOCK]PreToolUse(Bash)push 前に author 不正git rebase --exec 'commit --amend --reset-author'
[PUBLISH-AUTHOR]PreToolUse(Bash)公開操作前の author 通告F-1 フォーマットで識別子を提示するのみ
[PUBLISH-SAFETY] / [PUBLISH-SAFETY-FULL]PreToolUse(Bash)公開前の機密走査サブエージェントで 12 観点を走査・F-1 報告
[WORKTREE-REQUIRED]PreToolUse(Edit/Write)メインツリーでの編集を blockparallel-dev-worktree で worktree を作成
[FLOW-SELECT-REQUIRED]UserPromptSubmit実装系語を検出flow-feature / flow-maintenance 等を Skill 起動
[FLOW-SELECT-BLOCK]PreToolUse(Edit/Write)実装スキル未起動の編集を blockSkill 起動後に再試行
[PLAYWRIGHT-WORKTREE-REQUIRED]PreToolUse(MCP)メインツリーでの Playwright MCP を blockworktree に移動して再試行
[PROD-SKILL-READ-REQUIRED]PreToolUse(Bash) / UserPromptSubmit本番操作前のスキル未読-deploy-production を Skill 起動
[NO-DELEGATION] / [NO-DELEGATION-BLOCK]Stop / PreToolUse(Bash) / PostToolUse「〜してください」依頼形を検出能動文に書き直し、代行不可なら [NO-DELEGATION-ABORT]
[NO-DEFERRAL] / [NO-DEFERRAL-BLOCK] / [NO-DEFERRAL-RESPONSE]PreToolUse(Bash) / PostToolUse(Edit) / Stop「別 PR で対応」「将来課題」等の先送り検出本ターン内で完遂か中止の 2 択に書き換え
[TEXTLINT] / [TEXTLINT-BLOCK] / [TEXTLINT-CLEAN]PostToolUse(Edit/Write)textlint 違反検出 / 解消text-dictionary/rule.md の違反時手順で即修正、CLEAN 時は次工程へ
[LINT-REVIEW-NEEDED]PostToolUse(Edit/Write)auto-fix 不可の lint 残Edit で直接修正
[TEST-FAILED]PostToolUse(Bash)テスト失敗を検出失敗原因を特定して修正、2 連続で報告のみ
[AUTO-COMMIT]Stopworktree 内の未コミット変更 30 秒超auto-ship スキルで Phase 1〜5 を自走
代表ワークフロー: AUTO-COMMIT → auto-ship
[Stop hook]
  ├ worktree 内 / HEAD attached を確認
  ├ 直近 commit から 30 秒経過を確認
  └ [AUTO-COMMIT] を注入
       ↓
[Claude が Skill: auto-ship を起動]
  ├ Phase 1: grouping-commits でコミット
  │     └ issue-N ブランチは managing-github-issues（verify モード）で 1 コミット化
  ├ Phase 2: git push -u origin <branch>
  │     └ [GIT-AUTHOR-PUSH-BLOCK] 時は rebase --reset-author で自動補正
  ├ Phase 3: formatting-pr で PR 作成
  │     └ [TEXTLINT-BLOCK] / [NO-DEFERRAL-BLOCK] 時は本文を再整形
  ├ Phase 4: fixing-review-findings で警告・critical を修正
  │     └ 再レビュー → 自動マージまで完遂
  └ Phase 5: 完了報告（PR URL・merge SHA・残存指摘件数）
       ↓
[次の Stop で dirty なしを確認・自然停止]
代表ワークフロー: FLOW-SELECT
[UserPromptSubmit]
  ├ 「実装」「修正」「直して」等の語を検出
  └ [FLOW-SELECT-REQUIRED] + セッションフラグを touch
       ↓
[Claude が Skill: flow-feature / flow-maintenance / flow-docs / managing-github-issues（pick モード） を起動]
       ↓
[skill-log に発火行を追加]
       ↓
[PreToolUse(Edit/Write)]
  ├ skill-log の発火行を確認 → フラグ消費
  └ Edit/Write を通過させる
