アーキテクチャ概要

.claude の責務と agent-home との分担、静的・動的の二層構造を 1 枚で把握する。

← .claude 設定層
位置づけ

/Users//.claude は Claude Code が起動時に読み込むユーザーグローバル設定領域。プロジェクト固有の設定は <project>/.claude/ にあるが、本ポータルは PC 全体に効くグローバル側の ~/.claude を対象にする。

責務は次の 4 つに集約する。

  • 設定の供給: CLAUDE.md（全プロジェクト共通の指示）・settings.json（hook・permissions・env）・statusline.py（ステータスライン描画）
  • カスタムリソースの保管: agents/（カスタム subagent 定義）・skills/（カスタム SKILL.md）・plugins/（プラグイン本体とキャッシュ）
  • 履歴・状態の蓄積: sessions/・projects/・history.jsonl・logs/・file-history/
  • ワークスペース: plans/（中間プラン）・tasks/・uploads/・paste-cache/
agent-home との分担

.claude は Claude Code の標準ディレクトリ。一方、~/agent-home/ はユーザー独自に整備した「実装本体の置き場」で、hook スクリプト（tools/hooks/*.sh）・スキル本体（skills/<name>/SKILL.md）・テンプレートなどを保持する。.claude/settings.json は hook の登録口として ~/agent-home/tools/hooks/*.sh を参照する。

┌──────────────────────────────────────────┐
│ ~/.claude （Claude Code 標準）              │
│  ├ CLAUDE.md         ← グローバル指示       │
│  ├ settings.json     ← hook・perm・env 登録 │
│  ├ statusline.py     ← ステータス行         │
│  ├ skills/  agents/  plugins/              │
│  └ sessions/ projects/ logs/ ...           │
└────────────┬─────────────────────────────┘
             │ hook が参照
             ▼
┌──────────────────────────────────────────┐
│ ~/agent-home （実装本体）                   │
│  ├ tools/hooks/*.sh ← settings.json から発火 │
│  ├ skills/<name>/SKILL.md ← Skill 本体      │
│  ├ ai-management-portal/ ← 統合ポータル（本サイト）│
│  └ templates/                              │
└──────────────────────────────────────────┘

設計上の含意: .claude 側を編集するだけでは hook の挙動は変わらない。実装変更は agent-home 側の .sh・SKILL.md を編集する。

静的層と動的層

.claude 直下のディレクトリは「人間が編集する静的層」と「ランタイムが書き込む動的層」に分かれる。git で追跡するのは前者のみで、後者は .gitignore で除外する設計。

層性質代表ディレクトリ
静的層人間が編集／レビュー対象skills/ agents/ plugins/ rules/ config/
半動的層ツール起動で増減するが残しておくplans/ tasks/ backups/
動的層ランタイム生成・短寿命sessions/ projects/ logs/ cache/ file-history/ paste-cache/ uploads/ session-env/ shell-snapshots/ dev-flow-logs/ jobs/ daemon*

設計ドキュメント（本ポータル）は ~/agent-home/ai-management-portal/ に統合済み。~/.claude/docs/ は廃止。

起動シーケンス（概念）
[Claude Code 起動]
  ↓
[settings.json 読み込み]
  ├ permissions（deny / allow）
  ├ env（環境変数）
  └ hooks（PreToolUse / PostToolUse / Stop / SessionEnd 等）
  ↓
[CLAUDE.md 読み込み]
  └ §1〜§18 のグローバル指示が system prompt に追加
  ↓
[statusline.py 起動]
  └ 進捗バーをステータス行に描画
  ↓
[セッション開始]
  ├ skills/ から Skill が発見される
  ├ agents/ から subagent_type が解決される
  └ plugins/ から MCP・command が登録される
