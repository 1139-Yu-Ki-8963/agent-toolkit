基本ルール（番号なし）
  • 言語: 思考は英語・対話は日本語。コード識別子は英語維持
  • 日本語の文体: 句構造・英語直訳の回避・読み手との距離感の 3 観点
  • 対話: 不足時は 1 問ずつ確認。アスキーアート図示を優先
  • コーディング原則: YAGNI / KISS / DRY、複雑度抑制、後方互換シム禁止、根本原因主義

節テンプレート（共通形）

新規ルールを §N として追加する際は次のテンプレートに従う。

## N. ルール名（slug）

PostToolUse / PreToolUse / Stop フックが `[TAG]` を含む additionalContext を注入した場合は、必ず以下を実行する。

1. （Claude の手順）
2. （並列起動・サブエージェント指示など）
3. **再帰防止**: 同一対象に N 回連続発火した場合は事実報告のみ行い続行する
欠番について

§3・§10 は欠番。過去にあった節（タスク管理・他の自動化）の廃止に伴い番号を空けたまま運用している。再利用せず番号を温存する（既存 PR / コミット履歴での参照を壊さないため）。
