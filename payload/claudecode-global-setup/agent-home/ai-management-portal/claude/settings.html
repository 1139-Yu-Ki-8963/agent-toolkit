CLAUDE.md の構造

CLAUDE.md は次の 3 ブロックで構成する。

  1. 基本ルール（言語・対話・コーディング原則）
  2. hook 注入タグへの対応（タグ別の応答手順へのポインタ）
  3. メンテナンスノート（陳腐化防止・dead code 検出の指針）

番号付き節（§1〜§18）の詳細は CLAUDE.md ルール一覧 を参照。各節は次の 3 要素を備えるテンプレート化済み: 趣旨・発火条件（hook タグ・状況）・Claude の手順 + 再帰防止。

settings.json の主要キー
キー用途備考
permissions.deny禁止コマンドの正規表現対話ログイン（gh auth login 等）を block
permissions.allow無確認実行を許可するコマンド・MCP/fewer-permission-prompts で蓄積
envセッションに注入する環境変数API token 等を集約
hooks.PreToolUseツール実行前のチェックNAMING・AUTHOR・PUBLISH-SAFETY 等を注入
hooks.PostToolUseツール実行後の事後処理TEXTLINT・AMBIGUITY-AUTO-FIX 等
hooks.UserPromptSubmitユーザー発話受信時のフラグ立てFLOW-SELECT 検出など
hooks.Stop応答終了時の整合性チェックAUTO-COMMIT・NO-DELEGATION 等
hooks.SessionEndセッション終了時の要約・後始末session-summary 連携
statusLineステータスライン設定statusline.py を起動
statusline.py の責務

セッション中のステータス行を描画する Python スクリプト。.status.json から full-cycle-flow の進捗を読み取って進捗バーを表示する。

編集時の注意
  • settings.json は ~/.claude/.allow-direct-edit を touch するか、update-config スキル経由で編集する。FLOW-SELECT の例外パスなので block はされないが、変更は backups/ に自動退避する設計
  • CLAUDE.md の冒頭に <!-- textlint-disable --> を保持する（指示文の文体を textlint 違反として扱わないため）
  • 権限変更は permissions.deny を優先で固める。allow はあくまで「無確認実行」の許可で、deny を弱めない
