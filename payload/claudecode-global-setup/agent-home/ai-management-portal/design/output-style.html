§1Output styles の定義

Output style とは、.claude/output-styles/ 配下に Markdown ファイルを置くだけで有効になる、 システムプロンプト全体を置き換える設定。 役割・トーン・応答形式・行動規範をセッション単位で丸ごと差し替える。
  • CLAUDE.md が「プロジェクト規約を追補するユーザーメッセージ扱いの指示」であるのに対し、Output style は system prompt 自体を置き換える点で影響範囲が異なる。
  • セッション開始時に読み込まれ、/clear で再適用される。

§2他ツールとの違い

機能適用範囲読み込みタイミング
Output stylesシステムプロンプト全体（役割・トーン・応答形式）セッション開始時に読み込み、/clear で再適用
CLAUDE.mdプロジェクト規約・コードベース知識（ユーザーメッセージ扱い）システムプロンプトの後に読み込み
Subagents独立した system prompt・モデル・ツールセットタスクごとに個別適用

§4frontmatter 仕様

フィールド用途デフォルト値
nameスタイルの表示名ファイル名から自動生成
description/config のピッカーに表示する説明文なし
keep-coding-instructionsClaude Code 組み込みのコーディング指示（スコープ規律・コメント規約・検証手順）を保持するかfalse（boolean）
force-for-pluginプラグイン専用。有効時に自動適用false（boolean）

frontmatter テンプレート

テンプレート
--- name: <kebab-case-name> description: </config ピッカーに表示する説明文> keep-coding-instructions: <true | false> ---
frontmatter の注意点
  • keep-coding-instructions のデフォルトは false。既存のコーディング規律（スコープ規律・コメント規約・検証手順）を保持したまま行動規範だけを追加したい場合は明示的に true にする。

§5有効化手順

  • .md ファイルを上記いずれかのディレクトリに配置するだけで自動認識される。
  • /config コマンドの Output style メニューから選択して切り替える。
  • または settings ファイルに "outputStyle": "<name>" を直接指定する。

§6実例: fable-like

本ポータルの運用環境では ~/.claude/output-styles/fable-like.md をユーザースコープに配置済み。 上位モデル（Fable 5）の既定挙動を Sonnet/Opus 向けに移植したもので、keep-coding-instructions: true により 組み込みのコーディング指示を保持したまま、以下 8 項目の行動規範を追加する。

  • 結論先行 — 最初の一文で結果に答える
  • 即行動 — 情報が揃ったら計画を並べず実行する
  • 進捗の実証 — 主張をツール結果と突合し、未検証は明言する
  • スコープ規律 — 要求以上の機能追加・抽象化をしない
  • ターン終了規律 — 「これからします」で終えず、実行してから終える
  • 境界 — 説明・質問への応答は評価に留め、修正は依頼後に行う
  • 委譲 — 独立したサブタスクはサブエージェントに委譲する
  • 長時間運用 — 最終報告は経緯の続きでなく再着地として書く
配置済みインスタンスの一覧は Output styles 一覧 を参照。
