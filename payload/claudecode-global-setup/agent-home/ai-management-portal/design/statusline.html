§1Statusline の定義

Statusline とは、Claude Code がターミナル下部に表示するステータス行を描画する仕組み。 settings.json の statusLine キーで外部コマンドを登録し、 そのコマンドが標準入力から受け取った JSON を元に任意のテキストを標準出力へ返す。

§2設定方法

settings.json
"statusLine": { "type": "command", "command": "python3 ~/.claude/statusline.py" }
  • type: "command" は、外部コマンドの標準出力をそのままステータス行として描画するモード。
  • コマンドはターンごとに呼び出される。
  • 本環境での実体配置先: ~/.claude/statusline.py。

§3入力仕様（stdin JSON）

以下は本環境の ~/.claude/statusline.py が実際に読み取っているフィールド。

フィールド内容
model.display_name使用中のモデル表示名
session_idセッション ID
session_nameセッション名（未設定時は空）
cwd現在の作業ディレクトリ
context_window.used_percentageコンテキストウィンドウ使用率
rate_limits.five_hour.used_percentage5 時間レート制限の使用率
rate_limits.seven_day.used_percentage7 日レート制限の使用率
上記以外にも Claude Code 本体は追加フィールドを渡す場合があるが、本環境のスクリプトが参照しているのはこの範囲。

§4現在の実装（4 行構成）

行内容
Line 1モデル名 / 現在フォルダ / Git ブランチ
Line 2セッション ID（先頭 8 文字）/ セッション名
Line 3コンテキスト使用率・5 時間/7 日レート制限使用率（記号で可視化）・worktree 本数
Line 4full-cycle-flow の進捗（flow-status.json が存在する場合のみ、進捗バーを表示）

使用率の記号・色分け

使用率記号
0〜12%○
13〜37%◔
38〜62%◑
63〜87%◕
88〜100%●

色は 50% 未満で緑、50〜79% で黄、80% 以上で赤に変わる。worktree 本数は上限 5 に対し、4 で黄、5 以上で赤になる。

§5カスタマイズ時の注意

変更前に確認すること
  • worktree 上限（WT_MAX = 5）は ポート管理規約 の worktree 5 本上限と一致させている。変更時は両方を同期する。
  • flow-status.json が見つからない場合、4 行目は出力されない設計（None を返す）。
  • 標準出力の各行がそのままステータス行になるため、長すぎる行はターミナル幅で折り返される。
