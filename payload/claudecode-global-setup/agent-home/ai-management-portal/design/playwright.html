§1背景と問題

Playwright MCP はデフォルトで CWD 直下に .playwright-mcp/ ディレクトリを自動生成し、スクリーンショット・コンソールログ・ページスナップショットを書き出す。
未設定時のリスク
  • CWD はプロジェクトごとに異なるため、.playwright-mcp/ が各プロジェクト root に散在する
  • コンソールログ・ページスナップショットが無秩序に蓄積し、ディスクを圧迫する
  • .gitignore に入れ忘れると git 管理下に混入する

§2出力先の設計

ラッパースクリプト launch.sh が CWD から basename でプロジェクト名を取得し、 ~/agent-home/tools/MCP/playwright/<project>/ にプロジェクト別サブディレクトリを自動作成する。 新規プロジェクトでも追加設定は不要。
Playwright MCP サーバー │ ├─ launch.sh ラッパー │ CWD から basename でプロジェクト名を取得 │ ↓ │ ~/agent-home/tools/MCP/playwright/ │ ├── agent-home/ │ │ ├── console-*.log │ │ └── page-*.yml │ ├── -/ │ │ ├── screenshot-*.png │ │ └── console-*.log │ └── / │ └── ... │ └─ browser_take_screenshot の filename 引数 ↓ hook が許可パスを検査 ├── ✓ ~/agent-home/tools/MCP/playwright/** ├── ✓ $CLAUDE_JOB_DIR/tmp/* ├── ✓ */docs/**（明示 commit 用） └── ✗ それ以外 → [FILE-PLACEMENT-BLOCK]
パス 用途 指定方法
~/agent-home/tools/MCP/playwright/ MCP 自動出力 --output-dir
$CLAUDE_JOB_DIR/tmp/ セッション一時ファイル filename 引数
<repo>/docs/<feature>/screenshots/ commit 対象のドキュメント画像 filename 引数

§3設定方法

プラグイン設定

# ~/.claude/plugins/.../playwright/.mcp.json
{ "playwright": { "command": "/Users/MacPro/agent-home/tools/MCP/playwright/launch.sh", "args": [] } }

ラッパースクリプト

# ~/agent-home/tools/MCP/playwright/launch.sh
#!/usr/bin/env bash project="$(basename "$(pwd)")" output_dir="$HOME/agent-home/tools/MCP/playwright/$project" mkdir -p "$output_dir" exec npx @playwright/mcp@latest --output-dir "$output_dir" "$@"
CWD のプロジェクト名を取得し、サブディレクトリを自動作成する。新規プロジェクトでも追加設定不要。

方法 設定箇所 備考

§4hook による強制

check-playwright-filename.sh が mcp__playwright__browser_take_screenshot の filename 引数を PreToolUse で検査する。 --output-dir で制御される自動出力とは独立して、明示的な filename 指定を対象とする。

検査ロジック

  1. filename 引数がなければ通す（レスポンスのみで受け取るケース）
  2. 相対パスなら block（CWD 直下への書き込みになるため）
  3. 絶対パスで許可ロケーション外なら block

注入タグ

[FILE-PLACEMENT-BLOCK] Playwright MCP の filename 引数が file-placement 規約に違反しています。
hook は browser_take_screenshot のみを検査する。コンソールログやページスナップショットの自動出力は --output-dir で制御する。

§5新規プロジェクトへの適用

ラッパースクリプト launch.sh が CWD から自動的にプロジェクト名を判定し、サブディレクトリを作成する。 新規プロジェクトを追加しても個別設定は不要。

唯一必要な作業は、新規プロジェクトの .gitignore に .playwright-mcp/ を追加することだ。 ラッパースクリプトで出力先を自動振り分けしてあっても、CWD 直下に出力が落ちてしまうケース（MCP の挙動変更など）への安全ネットとして機能する。

プラグイン自動更新に注意
  • プラグインが自動更新された場合、.mcp.json が初期状態に戻る可能性がある
  • command が launch.sh を指しているか定期確認が必要
  • 確認方法: cat ~/.claude/plugins/.../playwright/.mcp.json で command を目視する
