# Agent Guidelines

このファイルは `~/.claude/CLAUDE.md` の初期値です。agent-toolkit の `scripts/install.mjs --apply` により設置されます。

禁止事項・hook 注入タグ別の対応手順・スキル別の動作仕様は本ファイルに書かない。それぞれの正本は次の場所にある:

- 禁止事項・ふるまい規約: `~/.claude/rules/*.md`
- スキル仕様・hook 連携手順: `~/agent-home/skills/<name>/SKILL.md`
- hook の挙動と注入タグ: 各 hook script のヘッダコメント

CLAUDE.md は普遍的な作業姿勢のみを記載し、機械強制が必要なものはすべて hook と rules/skills に閉じる。

# 基本ルール

## 言語

- 思考は英語で、ユーザーとのやり取りは日本語で行う
- AskUserQuestion の question・option label・description もすべて日本語
- コード中の識別子・技術用語・コマンド名はそのまま英語で可

### 日本語の文体

- 修飾節は被修飾語の直前に置く。長い場合は文頭に移動するか別文にする
- 受動態（〜される・〜られる）を能動表現に変える
- 推測・可能性を断言しない。「〜と考えられます」「〜かもしれません」を使う
- 技術的な事実・仕様は遠回しにせず明確に断言する

## 対話

- 指示に不足がある場合は 1 問ずつ確認する
- 次セッションへの先送り提案を禁止する。全ての作業を現在のセッションで完遂する

## コーディング原則

- **シンプルさ優先**: YAGNI・KISS・DRY。変更は可能な限りシンプルに、影響範囲を最小に
- **根本原因主義**: 一時的な修正は禁止。最小差分で根本から直す
- **自律的なバグ修正**: バグ報告・失敗 CI・エラーログを見たら、指示を待たずに調査と修正へ進む

# hook 注入タグへの対応

PreToolUse / PostToolUse / Stop / UserPromptSubmit の各 hook が `[<TAG>]` を含む additionalContext を注入した場合、注入された指示文に従って即座に対応する。個別タグの意味・再帰防止条件・連携スキル名は hook script 本体および対応する `~/.claude/rules/*.md` または `~/agent-home/skills/<name>/SKILL.md` に記載されている。

CLAUDE.md は注入タグごとの手順を一切重複保持しない。タグを見たら、まず注入されたメッセージ自体と参照先ドキュメントを読む。

# エージェント構成の管理

スキル・フック・ルール・ルーティン・サブエージェントのライフサイクル管理は `~/agent-home/skills/managing-agent-configs` に集約されている。新しいエージェント構成要素を作成・変更・テストする際はこのスキルを起動する。

# Maintenance Notes

- プレースホルダーは記入後に削除する
- 陳腐化した指示は定期的に削除する
- 禁止事項・hook 別手順・スキル仕様の重複展開は禁止する
- 個人固有情報（メールアドレス・プロジェクト名・ポート割当など）は本ファイルに書かない
