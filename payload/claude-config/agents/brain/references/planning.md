# Planning 参照資料

brain が Planning モードで使うテンプレートとチェックリスト。

## 作業指示の組み立てテンプレート

### worker-haiku 向け（機械的タスク）

```
対象: [ファイルパターン or 一覧]
変更パターン: [具体的な変換ルール]
成功条件: [grep パターン / ファイル数 / 構造チェック]
除外: [触らないファイル]
```

### worker-sonnet 向け（判断を伴うタスク）

```
目的: [何を達成したいか]
調査範囲: [ディレクトリ / ファイルパターン]
判断基準: [何を基準に判断するか]
出力形式: [期待するレポート構造]
```

### researcher 向け（外部情報収集）

```
調査対象: [何を調べるか]
情報源の優先順位: [公式ドキュメント > コミュニティ > ブログ]
深さ: [概要 / 詳細 / 網羅的]
出力形式: [期待するレポート構造]
```

## タスク分解のチェックリスト

1. タスクは1つの worker で完結するか？ → 複数なら分割
2. 判断が必要か？ → NO なら worker-haiku、YES なら worker-sonnet
3. 外部情報が必要か？ → YES なら researcher
4. 成功条件は機械的に検証可能か？ → YES なら具体パターンを定義

## リスク管理

- 変更対象ファイル数が多い場合、まず1ファイルで試行 → brain が検証 → 残りを一括
- 不可逆な変更（削除等）は成功条件を厳密に定義してから実行
