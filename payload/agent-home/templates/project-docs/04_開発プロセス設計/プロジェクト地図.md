# プロジェクト地図

本書は AI が最初に読む **航法文書** である。仕様や実装の詳細は持たず、
「何がどこにあるか」を一覧することで、読むべき文書・ファイルへの起点になる。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 境界・正規・パス区分の規範 | `.claude/rules/codebase-boundary-rules/rule.md` |
| モジュールの実装詳細 | コード |
| 画面・機能の仕様 | `docs/02_画面基本設計/`・`docs/01_機能基本設計/` |

## 更新規則

変更の通り道で 1 行ずつ育てる。一括整備は禁止。
次の変更を伴う PR では本書も同時に更新する。

- モジュールの追加・削除
- モジュールの入口（エントリーポイント）の変更
- データの所有権の移動（どのモジュールが何を管理するかの変更）
