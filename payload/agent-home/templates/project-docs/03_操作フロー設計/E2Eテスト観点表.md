# <フロー名> E2E テスト観点表

本書は複数システム・実環境・画面横断（large）の「何を検証すべきか（観点）」の正である。画面をまたぐユーザーシナリオの受け入れ条件として機能する。具体的な手順・アサーションはテストコードが正。

## テストサイズ対応表

| サイズ | 実行範囲 | 例 | 置き場 |
|---|---|---|---|
| small | 1 プロセス内で完結 | vitest・関数単位テスト | 各機能の単体テスト観点表 |
| medium | 1 マシン内（実ブラウザ・ローカル実 DB 可） | 単画面ブラウザ検証 | 各画面・各機能の結合テスト観点表 |
| large | 複数システム・実環境・画面横断 | 画面をまたぐユーザージャーニー | E2E テスト観点表（本書） |

所属は実行技術ではなく、対象が複数システム・画面をまたぐかで決める。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 単一画面の検証（medium） | 各画面の結合テスト観点表 |
| 関数・メソッド単位（small）の検証 | 各機能の単体テスト観点表 |
| 自動テストの具体的な手順・アサーション | テストコード |
| lint・静的解析 | Phase 7 完了チェック / hooks |

## 実行規則

- 関連画面すべての実装が完了した後に実行する
- 受け入れ ◯ の全 green はリリース条件
- CI で実行する
- 全行にテスト参照または手動参照が付き、受け入れ ◯ が全て green になったら frontmatter の `status: traced` に更新する
