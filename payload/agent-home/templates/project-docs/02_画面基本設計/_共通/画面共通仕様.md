# 画面共通仕様書

本書はプロジェクト全画面に共通する挙動の **正（権威ある仕様源）** である。
各画面設計書の §8（逸脱と例外）は本書を比較先として逸脱理由を記録する。

> 昇格規則: 複数の画面の §8（逸脱と例外）に同一の逸脱が並んだら、本書へ昇格して共通仕様とし、各画面の §8 から当該行を削除する（逸脱の多数派は共通の誤りを意味する）。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 数値・色・フォント | _共通/DESIGN.md |
| 画面固有の挙動 | 各画面設計書の該当章 |
| UI 文言 | _共通/メッセージ定義書.md |
| 実装の詳細（状態変数・DOM 構造） | 実装コード |

---

## §1 共通画面状態の規則

全画面に適用する 4 状態の標準仕様。各状態の「見せ方」の数値は _共通/DESIGN.md を参照する。

| 状態 | 標準の見せ方 | 操作可否 |
|---|---|---|
| loading | スケルトンまたはスピナーを表示する。ページ全体がブロックされる場合はインタラクション不可 | 原則禁止（例外は §8 に記録） |
| empty | 空状態のメッセージと次のアクションへの案内を表示する | 可（案内アクションのみ） |
| error | エラーの種別と対処方法を表示する。文言は _共通/メッセージ定義書.md を参照する | リトライ操作のみ可 |
| ready | 通常のインタラクション可能状態 | 全操作可 |

---

## §2 共通操作規則

### キーボード・フォーカス移動

| 規則 | 内容 |
|---|---|
| フォーカス移動 | Tab で前進・Shift+Tab で後退。インタラクティブ要素すべてに到達できること |
| フォーカストラップ | ダイアログ・モーダルが開いている間は背面要素にフォーカスが移らないこと |
| フォーカス可視性 | _共通/DESIGN.md のアクセシビリティ基準に従う |

### 確認ダイアログを出す標準条件

| 条件 | 内容 |
|---|---|
| 破壊的操作（削除・一括変更） | 操作前に確認ダイアログを出す |
| 入力内容の破棄 | フォームに未保存の変更がある状態でページを離れる場合は確認を出す |
| 取り消せない操作 | 確認ダイアログで「取り消せない」旨を明示する |

---

## §3 共通レイアウト原則

数値（幅・高さ・余白）は _共通/DESIGN.md を参照する。本節は構造の原則のみを持つ。

| 要素 | 原則 |
|---|---|
| ヘッダ | 全画面で共通レイアウト。画面固有のタイトルを表示する |
| ナビゲーション | 全画面で共通。現在位置を明示する |
| メインコンテンツ | ヘッダ・ナビゲーション以外の領域。各画面設計書の §2 が構造を定義する |
| フッタ | <プロジェクト方針に合わせて記述する> |
