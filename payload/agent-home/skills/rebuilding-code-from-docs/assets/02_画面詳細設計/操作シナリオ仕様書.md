# <画面名> 操作シナリオ仕様書（reverse-docs 版）

本書は画面詳細設計書 frontmatter の `operation_test_spec` キーから参照される。

本書は単一画面内でユーザー操作（フィルタ・ソート・切り替え等）を行い、操作後の状態をオリジナルコード環境とリバースコード環境で突合する（往復検証 L5）ための操作手順を定義する。画面をまたぐ複数画面シナリオ（large）は往復検証のスコープ外である（対応テンプレートなし）。単一画面内の操作シナリオは本書が担う。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 初期表示のみの検証（操作を伴わない画面状態） | 画面詳細設計書 frontmatter の `scenarios[]` |
| 抽象的な観点（何を検証すべきかの一般論） | 結合テスト観点表.md（往復検証観点表） |
| 画面をまたぐシナリオ（large） | 往復検証のスコープ外（対応テンプレートなし） |
| A-B 突合の実行機構・比較ロジック | syncing-reverse-env（本書は手順の入力のみを持つ） |
