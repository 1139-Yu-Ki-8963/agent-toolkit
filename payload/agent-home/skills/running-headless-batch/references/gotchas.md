# Gotchas（落とし穴集）

`running-headless-batch` の改修後の再検証にも使う、実機で確認済みの落とし穴。各項目は「事象・原因・対策」の3行構成。

## `--worktree` フラグの不存在

- **事象**: `claude --help` に `--worktree` という項目がない。公式ドキュメントの記載と実機が食い違って見える場面がある
- **原因**: claude CLI 2.1.206 時点でこのフラグは実装されていない
- **対策**: worktree（作業ツリーの分離複製）が必要な場合は、呼び出し側が `git worktree add` を事前に実行してから `claude -p` の cwd（作業ディレクトリ）をそこに向ける。実機 `claude --help` の出力を正とする

## バックグラウンドシェルの寿命

- **事象**: `run_in_background` で起動したシェルが、メインセッションの応答が返った直後に終了してしまう
- **原因**: Claude Code のバックグラウンドシェルは「メインセッションが最終結果を返してから数秒でシェル自体が終了する」仕様がある（適用範囲の詳細は未確定）
- **対策**: `nohup ... & disown` でシェルの生死からプロセスを切り離し、起動10秒後に `kill -0 $PID` で生存確認する。生存確認をせずに「起動した」と報告しない

## Bash サンドボックスの外向きネットワーク制限

- **事象**: `claude -p` の実行がタイムアウトまたは接続エラーで失敗する。実測では `Not logged in`（未ログイン）と表示されるケースがあるが、実際は認証済みで、API への接続がサンドボックスに拒否されているだけであり、認証切れと誤診断しないこと
- **原因**: `claude -p` は Anthropic API への外向き接続が必要だが、Bash ツールのサンドボックス（コマンド実行環境の権限制限機構）が許可ドメイン外への接続を拒否する環境がある
- **対策**: Phase 3（単発ドライラン）の時点でサンドボックス無効化の要否を判断し、必要ならユーザー承認を得る。無人ループ本番（Phase 4）に入ってから気付くと手戻りが大きい。サンドボックス無効化での再試行で切り分ける

## 終了コードは信用しない

- **事象**: `claude -p` が正常終了（exit 0）したのに、対象の作業が実際には未完了だった
- **原因**: ヘッドレス呼び出しは、途中で作業を打ち切っても正常終了コードを返すケースがある
- **対策**: 成否判定は必ずマーカーの実在チェック（`grep` 等）で行う。終了コードは判定に使わない

## CRLF・エンコーディングの混入

- **事象**: 対象一覧との文字列比較・存在確認が原因不明に失敗する
- **原因**: 対象一覧を機械生成した際に CRLF（Windows 由来の改行コード）や不揃いなエンコーディングが混入しやすい
- **対策**: Phase 2 の正規化（`tr -d '\r'`・エンコーディング統一・空行除去）を省略しない

## `~/.claude/projects/` の肥大化

- **事象**: 大量呼び出し後にディスク使用量が増え、Claude Code の起動が遅くなる
- **原因**: `claude -p` はデフォルトでセッション記録（JSONL 形式の対話ログ）を `~/.claude/projects/` に書き込み続ける
- **対策**: 大量呼び出しの既定として `--no-session-persistence` を必ず付ける

## `--permission-mode` 選択指針

- **事象**: ループが承認待ちで止まる、または想定外の操作まで許可してしまう
- **原因**: ヘッドレス実行では対話承認プロンプト自体が出ないため、`--allowedTools` で明示していないツールは拒否されて処理が止まる。逆に緩すぎるモードは意図しない操作まで許可する
- **対策**: 既定は `--permission-mode dontAsk` + `--allowedTools` の明示列挙。ファイル編集主体のタスクは `acceptEdits`。`bypassPermissions` は影響範囲を理解した上での最終手段とする

## Mac 本体のスリープ

- **事象**: 深夜に起動したループが翌朝止まっている
- **原因**: `claude -p` はローカル実行であり、Mac 本体がスリープ・シャットダウンするとプロセスごと停止する
- **対策**: 夜間運転前に本体のスリープ無効設定を確認する。クラウドで完結させたい場合は Routines（`~/agent-home/routines/` のクラウド定期実行の仕組み）が代替になる

## bash の連想配列非対応（macOS標準 bash 3.2）

- **事象**: `declare -A` を使ったループスクリプトが `bash: declare: -A: invalid option` で落ちる
- **原因**: macOS 標準搭載の `/bin/bash` はバージョン3.2系であり、連想配列（`declare -A`）に対応していない。`bash -c` がどの `bash` を解決するかは `PATH` 設定に依存する
- **対策**: `references/loop-design.md` の雛形は連想配列を使わず、`awk` によるファイルベースのカウンタ（`FAIL_COUNTS`）で失敗回数を管理する。改修時も連想配列を持ち込まない
