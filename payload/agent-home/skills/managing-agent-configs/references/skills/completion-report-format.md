# 完了報告フォーマット（正本）

全スキルの完了報告（ユーザーへの最終報告）の共通骨格。各スキルの SKILL.md は本ファイルを参照し、固有差分のみを宣言する（骨格のコピー禁止）。

## 共通骨格

````markdown
## 完了報告

**結論**: <何をどこまでやったか。80字以内・1文。「〜しました」で言い切る>

**成果物**(5行以内。6件以上は「代表3件 + 他N件」に圧縮):
- <パス> — <一言説明>

**検証**(実行した証拠のみ。未実行の項目は書かない):
| 確認内容 | 方法 | 結果 |
|---|---|---|
| <何を確認したか> | <コマンド/手段> | PASS / FAIL |

**未実施・制約**: <なければ「なし」1語。あれば1行ずつ・理由付き>
````

## 記入規定（読まれるための上限）

| 規定 | 値 | 理由 |
|---|---|---|
| 全体 | 20行以内 | 超えると流し読みされ、結論も読まれなくなる |
| 結論 | 80字以内・1文 | 通知プレビューやスクロール先頭で切れずに読める長さ |
| 成果物 | 5行以内 | 全列挙は diff の再掲。代表+件数で足りる |
| 検証の書き方 | 結果が先、手段は短く | 「PASS が何件か」だけ拾えれば流し読みに耐える |
| 禁止 | 経緯の時系列語り・ヘッジ表現（「おそらく完了」）・未実行項目の記載 | 報告は再着地であって作業ログではない |

長い作業の詳細は PR 本文・コミットメッセージ側に書き、チャット報告は本骨格に絞る。

## 2型の定義

| 型 | 対象 | 骨格との差分 |
|---|---|---|
| 作業報告型 | ファイル変更・構築・修正を行うスキル | 差分なし（共通骨格そのまま） |
| 調査報告型 | 成果物を作らない調査・レビュー系スキル | 「成果物」を「発見事項（3件以内・1行ずつ）」に読み替える |

この2型以外の派生は作らない。

## 拡張ルール（各スキルが持てる自由度）

- 可: 「検証」テーブルへの行追加（例: コミット数・CRITICAL/WARN 件数）。各スキルの SKILL.md の `## 完了報告` 節に箇条書きで宣言する
- 不可: 新セクション見出しの追加・別テンプレートファイルの新設・上限値の変更。骨格・見出し名・上限は本正本のみが変更できる

## 記入例（作業報告型）

````markdown

**結論**: スキル4件の整理（改名・rules統合・減量・参照修正）を2リポジトリでコミットし、PR #20・#6 をマージしました。

**成果物**:
- skills/generating-design-md/ — converting-html-to-design-md から改名、Phase 4 へ組み込み
- ~/.claude/rules/always/naming/commit-branch/ — 命名規約値と author-check.sh を統合
- 他: 参照追従26ファイル、リバース系スキル6件の削除

**検証**:
| 確認内容 | 方法 | 結果 |
|---|---|---|
| 旧名の残存なし | grep 全域 | PASS（0件） |
| author-check.sh 発火 | 実機3ケース | PASS |
| ポータル整合 | manage-portal.mjs verify | PASS（9/9） |

**未実施・制約**: ~/.claude の別セッション由来2ファイルは巻き込まず未コミットのまま。
````
