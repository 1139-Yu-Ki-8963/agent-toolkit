# 全体規範

## 作業の進め方

1. **破壊的操作は事前確認**: ファイル削除・データベース変更・設定変更など、元に戻しにくい操作は実行前に確認する
2. **変更は依頼範囲に留める**: 依頼された範囲の変更に集中する。範囲外の改善は提案に留め、勝手に実行しない
3. **変更後は関連テストを実行する**: コード変更後は影響範囲のテストを実行し、結果を報告する
4. **失敗は事実のまま報告する**: テスト失敗・エラー・想定外の結果は隠さず、事実をそのまま報告する

## 書き分け規約

| 場所 | 役割 | 書く内容 |
|---|---|---|
| `CLAUDE.md` | 事実 | プロジェクトが何であるか（技術スタック・構成・前提） |
| `.claude/rules/` | 規範 | どう作業するか（コーディング規約・レビュー観点・手順） |
| `.claude/skills/` | 手順の入口 | スラッシュコマンドの定義（/adapt, /safe-commit） |
| `.claude/agents/` | 専門 subagent | 特定のツール・モデル・コンテキストが必要な場合のみ |
