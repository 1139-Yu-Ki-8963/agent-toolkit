# テスト規範

1. **変更後の関連テストを実行し、実行範囲を報告する**: コード変更後は影響範囲のテストを実行し、何を実行したかを報告する
2. **期待値の書き換えは根拠がある場合のみ**: テストの期待値を変更する場合は、なぜ変更が正しいかの根拠を示す
3. **境界値と失敗系をテストする**: 正常系だけでなく、境界値・エラーケース・異常入力のテストを含める
4. **flaky テストは報告する**: テスト結果が不安定な場合は、そのテストを flaky として報告する
5. **フィクスチャにはダミー値を使う**: テストデータに本物の秘密情報・個人情報を使わない
