# バックエンド規範

1. **既存の API 設計に追従し、入力検証を必須とする**: エンドポイントの命名・レスポンス形式は既存に合わせ、すべてのユーザー入力を検証する
2. **例外を握りつぶさない**: catch ブロックで例外を無視しない。ログ記録またはリスローする
3. **構造化ログを使い、秘密情報をログに出さない**: ログは JSON 形式等の構造化された方法で出力し、トークン・パスワードを含めない
4. **タイムアウトとリトライを設定する**: 外部サービス呼び出しにはタイムアウトを設定し、リトライ戦略を明示する
5. **破壊的変更は影響を列挙して確認する**: API の破壊的変更は影響範囲を列挙し、事前に確認を得る
