# フロントエンド規範

1. **既存のコンポーネント設計・状態管理方式に従う**: 新しいパターンや状態管理ライブラリの持ち込みは提案に留め、既存方式を優先する
2. **アクセシビリティを確保する**: キーボード操作に対応し、フォーム要素には適切なラベルを付ける
3. **スタイル方式の混在を避ける**: CSS Modules・styled-components・Tailwind 等、プロジェクトで採用済みの方式に統一する
4. **文言をハードコードしない**: i18n の仕組みがある場合はそれに従い、文言を直接コードに埋め込まない
