# データベース規範

1. **migration の適用は確認必須。ロールバック不能な場合は明示する**: マイグレーションを実行する前に確認を得る。down マイグレーションが不可能な場合はその旨を明示する
2. **破壊的スキーマ変更は影響を列挙する**: カラム削除・型変更などの破壊的変更は、影響を受けるクエリ・機能を列挙する
3. **N+1 問題を指摘し、SELECT * を回避し、パラメタライズする**: クエリのパフォーマンス問題を発見したら指摘する。SELECT * は避け、SQL パラメータはバインド変数を使う
4. **トランザクション境界を意識する**: 複数のデータ操作が整合性を要する場合はトランザクションで囲む
