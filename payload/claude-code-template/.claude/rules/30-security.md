# セキュリティ規範

## 秘密情報の取り扱い

1. **コード・会話に秘密情報を含めない**: API キー・パスワード・トークンをコードや会話に直接書かない
2. **deny の回避を試みない**: permissions.deny で保護されたファイルへのアクセスを迂回しない。必要な場合は変数名だけを伝える
3. **発見時は値を写さず場所だけ報告する**: 秘密情報を発見した場合、値をコピーせずファイル名と行番号のみを報告する

## セキュリティレビュー観点

- **入力検証**: ユーザー入力は必ず検証する。SQL はパラメタライズドクエリを使う
- **認証と認可の区別**: 認証（本人確認）と認可（権限確認）を混同しない
- **ログに秘密情報を出さない**: ログ出力にトークン・パスワード・個人情報を含めない
- **依存追加は提案制**: 新しいパッケージの追加は提案に留め、タイポスクワッティングに注意する
- **外部送信は確認**: 外部 API へのデータ送信は事前に確認する

## ガードの位置づけ

hooks のガード（pre-tool-use.mjs）は事故防止の補助装置であり、セキュリティの主防衛線ではない。主防衛線は `settings.json` の `permissions.deny` と人間によるコードレビューである。
