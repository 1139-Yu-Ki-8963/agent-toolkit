# コーディング規約

## 命名規則

- 変数名・関数名はキャメルケース（`getUserName`）
- クラス名はパスカルケース（`UserService`）
- 定数は大文字スネークケース（`MAX_RETRY_COUNT`）
- ファイル名はケバブケース（`user-service.ts`）

## インデント・フォーマット

- インデントはスペース2つ
- 行末セミコロン必須（TypeScript）
- 1行は120文字以内

## コメント

- 関数には JSDoc 形式でコメントを付与する
- TODO コメントには担当者と期限を記載する（`// TODO(yamada): 2026-08 までに対応`）

## エラー処理

- try-catch は最小限のスコープで使用する
- エラーログには操作コンテキスト（ユーザーID・リクエストID）を含める
- ユーザー向けエラーメッセージと内部ログは分離する
