# 共通設計書

## 認証・認可

- 認証方式: JWT（アクセストークン + リフレッシュトークン）
- アクセストークン有効期限: 30分
- リフレッシュトークン有効期限: 7日
- 権限モデル: RBAC（ロールベースアクセス制御）

## ログ設計

- ログレベル: ERROR / WARN / INFO / DEBUG
- 構造化ログ（JSON形式）を標準出力に出力
- リクエストIDをヘッダー `X-Request-Id` で伝搬

## エラーコード体系

| コード範囲 | 意味 |
|---|---|
| 1000-1999 | 認証・認可エラー |
| 2000-2999 | バリデーションエラー |
| 3000-3999 | ビジネスロジックエラー |
| 9000-9999 | システムエラー |

## 共通レスポンス形式

正常時: `{ "data": {...}, "meta": { "requestId": "..." } }`
異常時: `{ "error": { "code": 1001, "message": "..." }, "meta": { "requestId": "..." } }`
