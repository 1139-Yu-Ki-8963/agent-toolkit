## §1 画面概要

### 1.1 基本情報

| 項目 | 値 |
|---|---|
| 画面名 | 注文一覧 |
| 目的 | 受注担当者が注文を検索・絞り込みし、詳細確認やCSV出力につなげる |
| ルート | /orders |
| 遷移元 | なし（起点画面） |
| 遷移先 | 注文詳細（行クリック時） |

### 1.2 業務上の位置づけ

受注業務フローの起点となる画面であり、受注担当者は本画面から日々の注文状況を把握し、個別の詳細確認や経理向けのCSV出力へ進む。

## §2 機能一覧

### 2.1 機能一覧表

| キー | 機能名 | 概要 | 優先度 |
|---|---|---|---|
| 検索-絞り込み | 注文検索 | 受注番号・顧客名・ステータス・期間で一覧を絞り込む | 必須 |
| 一覧-csv出力 | CSV出力 | 絞り込み結果をCSV形式でダウンロードする | 必須 |
| 一覧-詳細遷移 | 詳細画面遷移 | 行クリックで注文詳細画面へ遷移する | 必須 |

### 2.2 機能間の関係

CSV出力は検索・絞り込みの結果に対して実行されるため、検索条件が確定してから出力操作が有効になる。

## §4 業務ルール（条件付き：業務モード・権限制御がある画面のみ）

### 4.1 業務モードと切替条件

| モード | 切替条件 | UI の変化 |
|---|---|---|
| 通常表示 | 既定 | 全項目・全操作を表示 |
| 出力制限モード | 利用者が経理権限を持たない場合 | CsvExportButton を非表示にする |

### 4.2 業務判断ポイント

利用者の権限区分（経理権限の有無）によってCSV出力操作の可否が分岐する。

### 4.3 業務フロー内の前後画面との関係

起点画面のため前画面はない。行クリックにより注文詳細へ受注番号を引き渡す。

### 4.4 権限別の UI 制御

| ロール | CSV出力ボタン | 検索フォーム |
|---|---|---|
| 経理担当者 | 表示 | 表示 |
| 受注担当者 | 非表示 | 表示 |

## §5 状態管理

### 5.1 URL パラメータ

| パラメータ名 | 型 | 必須 | 用途 |
|---|---|---|---|
| status | 文字列 | △ | ステータス絞り込み条件を URL 経由で復元する |
| dateFrom | 文字列（日付） | △ | 注文期間の開始日を URL 経由で復元する |
| dateTo | 文字列（日付） | △ | 注文期間の終了日を URL 経由で復元する |

### 5.2 セッション・ストレージ参照

| キー名 | 種別 | 用途 |
|---|---|---|
| order-list-last-search | session | 直前の検索条件を一覧再訪時に復元する |

### 5.3 メイン画面の状態変数

| 変数名 | 型 | 初期値 | 役割 |
|---|---|---|---|
| searchKeyword | 文字列 | 空文字 | 受注番号・顧客名の検索キーワードを保持する |
| statusFilter | 文字列 | 空文字（全件） | 絞り込み対象のステータス区分を保持する |
| orders | 注文レコードの配列 | 空配列 | 検索結果の注文一覧を保持する |
| isLoading | 真偽値 | 偽 | 検索実行中かどうかを保持する |
| currentPage | 数値 | 1 | 表示中のページ番号を保持する |

### 5.4 サブコンポーネントの状態変数

| コンポーネント | 変数名 | 型 | 役割 |
|---|---|---|---|
| OrderSearchForm | dateRange | 開始日・終了日を持つ組 | 期間絞り込みの入力値を保持する |

### 5.5 派生値（計算値）

| 派生値名 | 算出ロジック概要 | 依存する状態変数 |
|---|---|---|
| totalCount | orders の件数を集計する | orders |
| hasCancelledRow | orders の中にキャンセル状態が含まれるか判定する | orders |

### 5.6 画面専用ストア（永続化）

該当なし（画面固有の永続化なし）

### 5.7 共有ストア参照

| ストア / コンテキスト | 参照キー | 用途 |
|---|---|---|
| AuthContext | userRole | 経理権限の有無を判定しCSV出力ボタンの表示可否を決める |

### 5.8 検索条件更新関数

updateSearchCondition(条件差分) — 引数に渡した項目のみ現在の検索条件へマージし、ページ番号を1に戻したうえで再検索を行う。

### 5.9 初期値・デフォルト値一覧

| 変数名 | 初期値 | 根拠 |
|---|---|---|
| currentPage | 1 | 一覧は常に1ページ目から開始する業務要件のため |
| statusFilter | 空文字（全件） | 初期表示では全ステータスを対象とする業務要件のため |

## §6 データフロー

### 6.1 初期表示シーケンス

1. URL パラメータ取得
2. ストア初期化（AuthContext から利用者権限を取得）
3. API 呼び出し（§7 参照）
4. データ変換（§6.5 参照）
5. 画面状態を ready に更新

#### 6.1.1 起動時初期化（URL パラメータ → ストア書き戻し）

status・dateFrom・dateTo の各 URL パラメータを searchKeyword 以外の検索条件ストアへ書き戻す。

### 6.2 依存連鎖

statusFilter が変化すると orders の再取得が発生する。

### 6.3 ユーザー操作確定時の処理フロー

1. バリデーション（§11 参照）
2. 確認ダイアログ表示（該当なし）
3. API 呼び出し
4. 成功時: 一覧を更新
5. 失敗時: エラー表示（§11 参照）

### 6.4 データ更新トリガーの分類

| トリガー | 種別 | 処理概要 |
|---|---|---|
| 検索実行ボタン押下 | ユーザー操作 | 現在の検索条件で注文一覧を再取得する |
| CSV出力ボタン押下 | ユーザー操作 | 現在の検索条件でCSV生成APIを呼び出す |

### 6.5 データ変換（API レスポンス → 画面表示用データ）

| 変換名 | 入力型 | 出力型 | 変換概要 |
|---|---|---|---|
| formatOrderRow | 注文APIレスポンス | 一覧表示用の注文行 | 金額を3桁区切り文字列へ、ステータスコードを表示ラベルへ変換する |

### 6.6 画面状態の遷移

| 状態 | 遷移条件 | UI の表示 |
|---|---|---|
| loading | 初期表示・再取得中 | ローディング表示 |
| empty | データ 0 件 | 空状態メッセージ |
| error | API エラー | エラーメッセージ（§11 参照） |
| ready | データ取得成功 | メインコンテンツ |
