# 注文一覧 単体テスト仕様書

本書は `../詳細設計/単体テスト観点表.md` の各観点キーに対応する、具体的な入力値・手順・期待結果（アサーション）を記述する。観点表は「何を検証するか」の正であり、本書は Phase 4（設計書からのTDD実装）の SPEC ステップで実際にテストコードへ落とし込む前の、具体的ケースの設計時契約である。

**実装後はテストコードが正になる**（§15 実装契約と同じ扱い）。実装後、本書の各行は「実装後のテストファイル参照」列にファイルパスを記入し、以後の変更はテストコード側で行う。本書の入力値・アサーションを実装後に手で書き換えてはならない（乖離があれば観点表の乖離分類で処理する）。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 抽象的な観点（何を検証すべきかの一般論） | `../詳細設計/単体テスト観点表.md` |
| 実装後のテストコードの変更管理 | テストコード自体 |
| 複数画面にまたがる結合的な検証 | `../詳細設計/結合テスト観点表.md` + `結合テスト仕様書.md` |
