メンバー編集 のテストケース

画面の主要な操作パターンについて、確認すべき手順と期待結果を一覧化する。

テストケース 分類 操作手順 期待結果 優先度
表示-既存データ 表示 既存メンバーの編集画面を開く 登録済みの情報がフォームに初期表示される 高
入力-必須検証 検証 必須項目を空欄にして保存する 必須項目のエラーメッセージが表示される 高
入力-メール形式 検証 不正な形式のメールアドレスを入力して保存する メールアドレス形式のバリデーションエラーが表示される 高
保存-正常 操作 全項目を正しく入力して保存する メンバー一覧画面に戻り、変更内容が反映される 高
保存-重複 異常系 既存メンバーと同じメールアドレスで保存する メールアドレス重複のエラーメッセージが表示される 中
キャンセル-未保存 画面遷移 項目を変更した状態で「キャンセル」ボタンをクリックする 変更内容が破棄され、メンバー一覧画面に戻る 低
