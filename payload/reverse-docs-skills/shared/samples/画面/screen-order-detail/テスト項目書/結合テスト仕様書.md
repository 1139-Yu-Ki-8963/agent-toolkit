# 注文詳細 結合テスト仕様書

本書は `../詳細設計/結合テスト観点表.md` の各観点キーに対応する、具体的な操作手順・入力値・期待結果（アサーション）を記述する。観点表は「何を検証するか」の正であり、本書は Phase 4（設計書からのTDD実装）の SPEC ステップで実際にテストコードへ落とし込む前の、具体的ケースの設計時契約である。

**実装後はテストコードが正になる**（§15 実装契約と同じ扱い）。実装後、本書の各行は「実装後のテストファイル参照」列にファイルパスを記入し、以後の変更はテストコード側で行う。手動観点（`../詳細設計/結合テスト観点表.md` の手動観点表）に対応するケースは、本書ではなく手動観点表の手順欄が正のままとする（本書は自動化する観点のみを対象とする）。

## 本書に書かないもの

| 書かない内容 | 正の置き場 |
|---|---|
| 抽象的な観点（何を検証すべきかの一般論） | `../詳細設計/結合テスト観点表.md` |
| 手動観点の手順 | `../詳細設計/結合テスト観点表.md` の手動観点表 |
| 単一画面内操作の往復検証（L5）手順 | `操作シナリオ仕様書.md` |
| 実装後のテストコードの変更管理 | テストコード自体 |
