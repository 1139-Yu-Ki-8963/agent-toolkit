# 詳細ページ設計カタログ

## 出自

本文書は、リバース設計ポータル配下の全詳細ページを 5 つの再利用テンプレートに整理した設計カタログである。新しいページ種別が増えても、この 5 型のいずれかに割り当てるだけで統一感と実装の使い回しが保たれる。2026-07-16 にユーザー承認済みのカタログ（モック HTML）から転記した。

## テンプレート対応表

| テンプレート | 使い回すページ | 共通スロット構成 |
|---|---|---|
| T1 一覧型 | 画面一覧・API一覧・バッチ一覧・テーブル一覧・帳票一覧・外部連携一覧・機能一覧（7 種） | パンくず → 見出し → 検索 + 種別フィルタ → 明細表 → 件数注記 |
| T2 対訳辞書型 | 用語辞書 | パンくず → 見出し → 検索 + 分類チップ → 対訳表（用語 ⇔ コード識別子） |
| T3 属性表型 | 技術スタック・`解析サマリ`（計測値 + 規模） | パンくず → 見出し → 要約タイル列 → 明細表（出所・計算条件の列を必須） |
| T4 図解型 | 画面遷移図・ER図 | パンくず → 見出し → 凡例 → SVG 図（横スクロール枠内）→ 補足表 |
| T5 手順型 | 環境・実行手順（RUNBOOK 由来のページ全般） | パンくず → 見出し → 前提ツール表 → 番号付き手順表 → 割当表（ポート等） |

キーは連番禁止。テンプレート名は内容を要約した意味語（T1 一覧型 等）で付ける。「T」に続く番号は分類上の識別用途であり、行の管理 ID ではない。

## T1 一覧型

画面・API・バッチ・テーブル・帳票・外部連携・機能の 7 一覧が共用する、ユニットの棚卸し表。列構成だけが種別ごとに差し替わり（画面ならルート・コンポーネント、API ならメソッド・パス）、枠組みは全種別で同一である。

| 項目 | 内容 |
|---|---|
| 目的 | 対象種別の全ユニットを一望し、検索・フィルタで目的の行へ素早く到達できるようにする |
| 共通スロット | パンくず / 見出し / 検索ボックス + 種別フィルタチップ / 明細表 / 件数注記 |
| 差し替えスロット | 見出し語 / 表の列定義 / 行データ / フィルタの分類軸（任意） |
| 使い回し先 | 画面一覧・API一覧・バッチ一覧・テーブル一覧・帳票一覧・外部連携一覧・機能一覧 |

## T2 対訳辞書型

用語辞書が使用する語彙の対訳表。T1 との違いは「分類チップでの絞り込み」と「用語 ⇔ コード識別子の対応列」が必須であることの 2 点である。

| 項目 | 内容 |
|---|---|
| 目的 | 業務用語・技術用語・略語とコード上の識別子を対応づけ、読み手が用語からコードへ、コードから用語へ双方向に辿れるようにする |
| 共通スロット | パンくず / 見出し / 検索ボックス + 分類チップ / 対訳表（用語・定義・コード上の対応） |
| 差し替えスロット | 分類軸（業務 / 技術 / 略語）/ 行データ |
| 使い回し先 | 用語辞書 |

## T3 属性表型

技術スタック・`解析サマリ`（計測値 + 規模）が共用する、概要タイル + 明細表の 2 段構成。明細表には数値・記載の出所列（定義ファイル・計算式）を必ず置き、事実の検証可能性を担保する。

| 項目 | 内容 |
|---|---|
| 目的 | 集計値・属性値を一覧しつつ、各値がどこから得られたか（定義ファイル・計算式）を読み手が検証できるようにする |
| 共通スロット | パンくず / 見出し / 要約タイル列 / 明細表（出所・計算条件の列を必須） |
| 差し替えスロット | タイルの項目 / 表の列定義 / 出所列の中身（ファイルパス or 計算式） |
| 使い回し先 | 技術スタック・`解析サマリ` |

## T4 図解型

画面遷移図・ER図が共用する、凡例 → SVG 図 → 補足表の 3 段構成。図は横スクロール枠に収め、ページ全体の幅を壊さない。図の要素から該当設計書へリンクする。

| 項目 | 内容 |
|---|---|
| 目的 | 構造・関係性を図で直感的に把握させつつ、図だけでは伝わらない詳細（トリガー・カーディナリティ等）を補足表で補う |
| 共通スロット | パンくず / 見出し / 凡例 / SVG 図（横スクロール枠内）/ 補足表 |
| 差し替えスロット | 凡例の記号定義 / SVG 本体 / 補足表の列定義 |
| 使い回し先 | 画面遷移図・ER図 |

## T5 手順型

環境・実行手順（RUNBOOK 由来のページ全般）が使用する、前提 → 手順 → 割当の 3 表構成。手順表の番号は実行順という意味を持つ（構造としての番号であり、意味キー規約の対象外）。

| 項目 | 内容 |
|---|---|
| 目的 | 環境構築・起動手順を、前提ツールの確認から実行順序、割当値（ポート等）まで一貫して読める形で提示する |
| 共通スロット | パンくず / 見出し / 前提ツール表 / 番号付き手順表 / 割当表（ポート等） |
| 差し替えスロット | 前提ツール表の行 / 手順表の行 / 割当表（ポート等）の行 |
| 使い回し先 | 環境・実行手順 |

## 運用ルール

新しい詳細ページを追加するときは、まずこの 5 型への割り当てを決める。どれにも当てはまらない場合のみ第 6 のテンプレートを起案し、このカタログへ追記する。

## 範囲外

- 新ページ生成スキル 5 本（用語辞書 T2 / 技術スタック T3 / 画面遷移図 T4 / ER図 T4 / 環境実行手順 T5）の実装は本カタログの対象外。ポータル側の受け口（FUTURE_PAGES）に該当 HTML を配置すれば、生成スキルが未実装でもカードとして反映される
- unit-list テンプレ群のデザイントークン共通化・テーマ対応（T1 統一）は本カタログの対象外
