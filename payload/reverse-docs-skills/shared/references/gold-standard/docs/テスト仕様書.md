# アイテム一覧 テスト仕様書

## 正常系シナリオ

### シナリオ: 一覧表示→検索→詳細遷移

| ステップ | 操作 | 期待結果 |
|---|---|---|
| 画面表示 | /items にアクセス | CircularProgress 表示後、アイテム一覧テーブルが表示される |
| 初期ソート確認 | — | items が updatedAt 降順（SortDirection.Desc）でソートされている |
| 検索実行 | キーワード欄に「テスト」入力 → 「検索」ボタン押下 | fetchItemList({ keyword: 'テスト', limit: 20, offset: 0 }) が実行され、結果がテーブルに反映 |
| 詳細遷移 | テーブル行をクリック | /items/:id へ遷移する |
| 新規追加遷移 | 「新規追加」ボタン押下 | /items/new へ遷移する |

### シナリオ: 削除

| ステップ | 操作 | 期待結果 |
|---|---|---|
| 削除実行 | 「削除」ボタン押下（event.stopPropagation で行クリック抑制） | deleteItem(id) 実行後、items から対象行が filter で除去される |

## 異常系シナリオ

### シナリオ: API失敗

| ステップ | 操作 | 期待結果 |
|---|---|---|
| 初期表示失敗 | API応答 !response.ok | setError でエラーメッセージ設定 → Typography(color="error") に表示 |
| 検索失敗 | 検索実行 → API応答 !response.ok | window.alert('検索に失敗しました') |
| 削除失敗 | 削除実行 → API応答 !response.ok | window.alert('削除でエラーが発生しました') |

## クリーンアップ確認

| 確認項目 | 期待動作 |
|---|---|
| useEffect アンマウント | cancelled=true で後続の setState が抑制される |
| finally ブロック | setLoading(false) が成功・失敗両方で実行される |
