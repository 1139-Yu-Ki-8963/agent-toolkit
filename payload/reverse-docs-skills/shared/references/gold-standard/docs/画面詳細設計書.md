# アイテム一覧 画面詳細設計書

## §1 画面概要

アイテムの一覧表示・検索・新規追加への遷移・削除を提供する画面。

## §2 機能一覧

| キー | 機能名 | 概要 |
|---|---|---|
| 一覧表示 | アイテム一覧表示 | 登録済みアイテムをテーブル形式で表示する |
| 検索 | キーワード検索 | キーワードで絞り込み検索を実行する |
| 新規追加 | 新規追加遷移 | 新規登録画面（/items/new）へ遷移する |
| 削除 | アイテム削除 | 選択したアイテムを削除する |
| 詳細遷移 | 詳細画面遷移 | 行クリックで詳細画面（/items/:id）へ遷移する |

## §3 画面構造

ルート要素 `Box` の中に以下の構造を持つ:
- 見出し `Typography` (h1): 「アイテム一覧」
- 検索フォーム `Box` (form): TextField + Button（検索）+ Button（新規追加）
- エラー表示 `Typography` (color="error"): 条件={error}
- ローディング `CircularProgress` (size={24}): 条件={loading}
- テーブル `TableContainer` (component={Paper}): Table > TableBody > ヘッダ行 + データ行

## §7 API通信仕様

### §7.1 一覧取得

- エンドポイント: POST /api/graphql
- リクエスト: `{ query: ITEM_LIST_QUERY, variables: { keyword, limit, offset } }`
- レスポンス: `body.data as ItemListResponse`
- ヘッダ: `{ 'Content-Type': 'application/json' }`
- エラー時: `throw new Error('一覧の取得に失敗しました（status: ${response.status}）')`

### §7.2 削除

- エンドポイント: POST /api/graphql
- リクエスト: `{ query: DELETE_ITEM_MUTATION, variables: { id } }`
- ヘッダ: `{ 'Content-Type': 'application/json' }`
- エラー時: `throw new Error('削除に失敗しました（status: ${response.status}）')`

## §8 イベント処理

| キー | ハンドラ名 | 発火要素 | 挙動 |
|---|---|---|---|
| handler-fetchItemList-API | fetchItemList | プログラム呼出し | GraphQL で一覧取得。!response.ok で throw |
| handler-deleteItem-API | deleteItem | プログラム呼出し | GraphQL で削除。!response.ok で throw |
| handler-handleSearchSubmit-検索 | handleSearchSubmit | form の onSubmit | event.preventDefault → fetchItemList({ keyword: form.keyword, limit: PAGE_SIZE, offset: 0 }) → setItems。catch で window.alert。finally { setLoading(false) } |
| handler-handleAddClick-遷移 | handleAddClick | 「新規追加」ボタン onClick | navigate('/items/new') |
| handler-handleRowClick-遷移 | handleRowClick | TableRow の onClick | navigate(`/items/${id}`) |
| handler-handleDeleteClick-削除 | handleDeleteClick | 「削除」ボタン onClick | deleteItem(id) → filter で items から除去。catch で window.alert |
| handler-sortItemsByKey-ソート | sortItemsByKey | プログラム呼出し | スプレッド構文でコピー → sort → Desc なら reverse。useEffect内での呼出し引数: (result.items, 'updatedAt', SortDirection.Desc) |
| effect-load-keyword変更 | useEffect([form.keyword]) | マウント・keyword変更 | let cancelled=false → setLoading(true) → setError(null) → fetchItemList({ keyword: form.keyword, limit: PAGE_SIZE, offset: 0 }) → sortItemsByKey(result.items, 'updatedAt', SortDirection.Desc) → setItems。catch: setError(message)。finally: setLoading(false)。cleanup: cancelled=true で setState を抑制 |

## §9 領域別仕様

### 検索フォーム

- `TextField`: label="キーワード" size="small" value={form.keyword} onChange で setForm
- `Button` (submit): variant="contained" 文言「検索」
- `Button` (add): variant="outlined" onClick={handleAddClick} 文言「新規追加」

### データテーブル

- ヘッダ行: columns.map で `HeaderCell`（key={column.key} sx={{ width: column.width }} 文言={column.label}）+ 操作列（HeaderCell sx={{ width: 80 }} 文言「操作」）
- データ行: items.map で `TableRow`（key={item.id} onClick={handleRowClick(item.id)}）> TableCell×5（code, name, category, quantity, updatedAt）+ 削除ボタン（Button size="small" color="error" onClick={stopPropagation + handleDeleteClick} 文言「削除」）
