# アイテム一覧 コーディング規約（実装模範10観点）

盲検再構築の入力として使用する。本コーディング規約に従ってコードを書くこと。

| 観点 | 規約 |
|---|---|
| 宣言形式 | 型定義の折返し: プロパティが3個以上なら1行1プロパティ |
| useState型引数 | useStateの型引数を常に明示する（`useState<string \| null>(null)` 等） |
| catch形式 | `catch (err)` で受け、`err instanceof Error ? err.message : 'デフォルト'` で型ガード |
| if/JSX属性の真偽判定 | if文は素のtruthiness（`if (!cancelled)`）、JSX属性は `Boolean()` |
| named import折返し | import シンボルが4個以上なら `{ }` 内で改行 |
| カリー本体形式 | アロー関数のカリーは `const f = (a) => (b) => ...` 形式 |
| オブジェクトリテラル/JSX改行 | プロパティ3個以上なら1行1プロパティ。JSX属性も同様 |
| JSX属性順 | key・ref 先頭 → 表示系（variant, size, label）→ 検証系（value）→ イベント系（onClick, onChange） |
| 宣言順 | モジュールレベル: `const` アロー → `function` 宣言。コンポーネント内: `const` アロー |
| 戻り値型注釈 | 関数の戻り値型を明示する（`Promise<ItemListResponse>` 等） |
