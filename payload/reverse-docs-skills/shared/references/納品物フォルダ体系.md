# 納品物フォルダ体系

本ファイルはリバース設計の納品物（成果物一式）の目録の正本である。各成果物を生成するスキルとの対応と実装状況をここで管理する。

画像からの文字起こし。リバース対象ユニットの成果物置き場（`<docs_root>` 配下）の全体構成と、各ファイルを生成するスキルの対応を示す。

実装状況の凡例: 【済】= 現行スキルが生成する成果物。【段階計画 Cycle 1〜3】= API・テーブル・バッチ・帳票・外部連携の詳細設計書群（未実装）。【段階計画 Cycle 4】= 上位抽象化スキルが担う基本設計・要件定義文書群（未実装）。

## 検証記録（verification/）

検証記録は納品物ではないため `docs/` の外に配置する。`docs/` と同階層の `verification/` フォルダに格納する。

```
verification/
└── screen-<画面ID>/
    ├── facts/
    │   └── <run_id>/
    │       ├── facts.yml
    │       ├── facts.lock
    │       └── recount-report.txt
    ├── 単体-<対象ファイルbasename>/
    │   └── <timestamp>/
    │       └── (単体検証記録)
    └── <timestamp>/
        ├── 修正指示書.md
        ├── 最終報告.md
        └── (テストログ・差分レポート等)
```

docs 内に検証記録は一切存在しない。facts・単体検証記録・修正指示書・最終報告・テストログはすべて verification/ 配下に配置する。

## 出典

`納品物フォルダ体系.md`（提供されたスクリーンショット2枚）の文字起こし。文字起こしのみを目的とし、実際のディレクトリ・ファイルの生成は行っていない。実装状況（【済】/【段階計画 Cycle N】）は文字起こし後にリポジトリの現行スキル構成と突合して追記したもの。
