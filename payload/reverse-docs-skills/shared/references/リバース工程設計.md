## NG帰着3系統

判定（Phase 7 / Step 27）でFAILした場合、原因を以下の3系統いずれかに帰着させ、該当スキルの改訂へ差し戻す。

| 系統 | 原因 | 差し戻し先 |
|---|---|---|
| (a) 執筆規律不足 | 詳細設計書の執筆規律・転記精度に起因する不一致 | authoring-screen-docs-from-code の執筆規律 reference を改訂 |
| (b) facts欠落 | 事実抽出プロファイルが対象コードの挙動を捕捉できていない | extracting-unit-facts-from-code の抽出プロファイルを改訂 |
| (c) 共通文書欠落 | 共通設計書・規約4種等のプロジェクト共通文書に該当挙動の記載が無い | compiling-project-common-docs を mode=append で起動 |
