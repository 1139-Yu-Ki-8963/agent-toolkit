§1概要

本スキルは「作業者」であり、設計書がまだ無い画面を対象に既存コードを調査してモックAPIを適用し、ログイン後まで画面表示できる状態（開通）にする。

責務の境界: 開通完了後の記帳・基準タグ確立は行わない。それは管理者が担い、orchestrating-reverse-docs-flow が syncing-reverse-env をレジストリモードで起動する。本スキルは他スキルを呼び出さず、他スキルの設定ファイルに書き込まない。
既存4子スキルと同格: generating-screen-list-for-reverse-docs / syncing-reverse-env / rebuilding-screen-unit-from-docs / rebuilding-code-from-docs と同じく、管理者スキル orchestrating-reverse-docs-flow が起動する完全仲介方式の子スキル。

§2使用タイミング

  • 対象画面にまだ設計書が無く、既存コードがログインを要求するなどの理由でそのままでは動作確認できないとき
  • 設計書が既にある画面はこのスキルの対象外（rebuilding-screen-unit-from-docs / rebuilding-code-from-docs を使う）

§3起動引数

引数内容
system対象システム名
screen_id対象画面ID
reverse_worktree対象画面のソースコードを持つ worktree の絶対パス
ports動作確認に使うポート情報
docs_root今後の設計書配置ルート（design_doc_path の組み立てに使用）

全量が管理者から渡される。単独起動時はユーザーが同じ引数を手渡せば動く。

§4実行手順（Phase 1〜4）

Phase 1: 入力ロード + preflight
  起動引数(system/screen_id/reverse_worktree/ports/docs_root)を受け取る
  いずれか欠落→status=ERRORで管理者へ差し戻し
  reverse_worktree配下で対象画面のソースコード一式の実在をGrep/Globで確認
  完了条件: 対象画面のソースコード一式の所在が確認済み

Phase 2: 既存コード調査
  ルーティング定義・認証ガード（ログイン必須化ロジック）・外部API呼び出し箇所を調査
  オリジナルコードの改変はまだ行わない（調査のみ）
  完了条件: モックAPI適用が必要な箇所の一覧が記録されている

Phase 3: モックAPI適用 + 動作確認
  特定箇所に最小限のモックAPI（固定レスポンスのスタブ等）を適用
  reverse_worktree上でログイン後まで画面表示を確認（実機 or 静的確認）
  変更をコミット（メッセージ:「【機能追加】<画面名>をモックAPIで開通」）
  完了条件: 画面がログイン後まで表示可能であることを確認済み、変更がコミット済み

Phase 4: 証跡保存 + 返却
  source_ref（開通確認時点のコミットハッシュ）
  verification_url（表示確認できたURL。未実施なら「未実施」と明記）
  design_doc_path（docs_root起点の <system>-<screen_id> 相当パス）
  完了条件: 返却ブロック（status・source_ref・verification_url・design_doc_path）が確定

§5完了条件

Phase完了条件
Phase 1対象画面のソースコード一式の所在が確認済み
Phase 2モックAPI適用が必要な箇所の一覧が記録されている
Phase 3画面がログイン後まで表示可能であることを確認済み、変更がコミット済み
Phase 4返却ブロックが確定している
Goal対象画面がモックAPI経由でログイン後まで表示可能になり、返却ブロック（status=UNLOCKED・source_ref・verification_url・design_doc_path）が管理者に渡っている

§6サブエージェント委任仕様

該当なし。本スキルは他スキル・サブエージェントを呼ばず、単独でPhase 1〜4を完結させる。allowed-tools に Agent・Skill・AskUserQuestion を含まない。

§7重要な注意事項

  • 本スキルは他スキル（syncing-reverse-env 等）を一切呼ばない。呼び出し・環境同期・基準タグ確立は管理者（orchestrating-reverse-docs-flow）が担う
  • 他スキルの設定ファイル（syncing-reverse-env/config.yml 等）に書き込まない
  • 開通済み画面の記帳（画面レジストリへの登録）は行わない。返却ブロックで管理者に情報を渡すのみ
  • 責務は「画面の開通まで」。基準タグ確立・往復検証は本スキルの対象外

§8Gotchas

  • 本スキルは「設計書がまだ無い」段階の画面が対象。設計書がある画面は rebuilding-screen-unit-from-docs / rebuilding-code-from-docs の対象であり本スキルは使わない
  • モックAPIの適用範囲は「ログイン後まで表示可能にする」最小限に留める。往復検証相当の精緻な動作確認は行わない（それは後続工程の責務）
