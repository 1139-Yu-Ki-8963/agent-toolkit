ローカル環境調査スキル
surveying-local-environment

実行環境の OS・パッケージ管理ツール・必要ツールの有無を調査し、env-config.json を生成するスキルの確定仕様。

対応 OSWindows WSL2 環境
検証状況surveying-local-environment スキルとして独立動作確認済み
依存bash、jq
関連資料orchestrating-reverse-docs-flow SKILL.md（Phase 4A から起動）
目次
  1. 概要
  2. 出力

§1概要

本スキルは、実行対象 PC の OS 種別・パッケージ管理ツール・開発ツール（cloc／node／python3／jq／git）のインストール有無を調査し、env-config.json に出力する。他のスキル（counting-code-lines 等）はこの env-config.json を読み込み、ツールの有無に応じて処理方式を分岐する。

cloc が未インストールの場合は計測精度が低下するため、パッケージ管理ツールに応じたインストールコマンドを組み立てて案内する。既に env-config.json が存在する場合は再調査せず、削除してからの再実行を促して終了する。

── 処理の全体像 ──────────────────────────────

  output_dir（出力先ディレクトリ）
       │  Phase 1  出力先の確認（env-config.json 既存なら削除案内で終了）
       ▼
  実行環境（uname / command -v 等）
       │  Phase 2  環境調査（OS・アーキテクチャ・パッケージ管理ツール・各種ツール有無）
       ▼
  env-config.json（Phase 3）
       │  Phase 4  cloc 未インストール時のインストール案内（該当時のみ）
       ▼
  他スキル（counting-code-lines 等）からの参照

§2出力

スキルを実行すると、以下の 1 点が生まれる。

env-config.json（output_dir 直下）
OS 種別 os・アーキテクチャ arch・検出したパッケージ管理ツール pkg_manager、cloc／node／python3／jq／git それぞれの真偽値を持つ tools、cloc 未インストール時のインストールコマンドを持つ install_commands、調査時刻 surveyed_at を持つ構造化 JSON。読んだ人はこの 1 ファイルだけで、実行環境がどの計測方式（cloc／フォールバック）を選べるかを判断でき、後続スキルはこれを読んで自動的に処理方式を切り替えられる。
