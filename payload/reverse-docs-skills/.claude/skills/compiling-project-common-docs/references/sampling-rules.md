# 層化サンプリング規則（Phase 2 の詳細手順）

`SKILL.md` の Phase 2（層化サンプリング）が参照する具体的な選定手順。Phase 2 本文で
述べた完了条件（サンプル記録.md にコマンドと選定ファイル一覧が記録済み）はここでは
繰り返さない。ここでは「層をどう定義するか」「層あたり何件選ぶか」「どう決定的に
選ぶか」「記録様式」に絞る。

## 層の定義

層（stratum）は、アーキテクチャ調査書の「ディレクトリ責務マップ」節の行をそのまま
使う。調査書が既に対象リポジトリを走査し、ディレクトリごとの責務を確定済みである
ため、本スキルが独自に層を再定義することはしない。ディレクトリ責務マップに無い
ディレクトリ（調査書執筆後に追加された等）は層に含めない。

| 判定 | 扱い |
|---|---|
| ディレクトリ責務マップに行がある | 1行 = 1層としてサンプリング対象にする |
| ディレクトリ責務マップに行が無い | 層に含めない（調査書の対象外はサンプリングの対象外） |
| 責務マップの行が示すディレクトリが空（ファイル0件） | 層として数えるが k=0 件でサンプル記録に「対象ファイル0件」と明記する |

## 層あたり k 値の決め方

層内ファイル数（`find <層のディレクトリ> -type f | wc -l` の実測値）を N とする。

```
k = N の平方根を切り上げた整数
k が 3 未満なら 3 に、10 を超えるなら 10 に丸める
N が k 未満（層内ファイル数が k に満たない）ならその層は全数（N件）を採用する
```

| N（層内ファイル数） | k（採用件数） |
|---|---|
| 1〜9 | N 件全数（k下限3に対しN<3なら全数採用でよい。N>=3ならmax(3, ceil(sqrt(N)))だが上限3のためN<=9は3件） |
| 9より大きく100以下 | `ceil(sqrt(N))`（3以上10以下の範囲に収まる） |
| 100超 | 10件（上限） |

## 決定的選択手順

乱数・シャッフルを禁止する。同一入力に対して常に同一のサンプル集合が再現される
手順のみを使う。

```bash
find <層のディレクトリ> -type f | sort | head -n <k>
```

- `find` の出力を必ず `sort` してから `head` で先頭 k 件を取る（`find` の出力順は
  ファイルシステム依存で非決定的なため、`sort` を省略しない）
- 除外パターン（`node_modules`・ビルド成果物・lockファイル等）がある場合は
  `find <層のディレクトリ> -type f | grep -vE '<除外パターン>' | sort | head -n <k>`
  の形で除外を先に適用してから決定的に選ぶ。除外パターンもサンプル記録に明記する
- 同じ層に対して二度実行しても同じ k 件が選ばれることを、サンプル記録に書く前に
  実際にコマンドを再実行して確認する

## サンプル記録.md の記載様式

各層ごとに、実行した選定コマンドと選定結果を漏れなく記載する。

```markdown
## 層: <ディレクトリ責務マップの行名>

- 対象ディレクトリ: `<相対パス>`
- 層内ファイル数（N）: <実測値>
- 採用件数（k）: <実測値>
- 選定コマンド: `find <層のディレクトリ> -type f | sort | head -n <k>`
- 選定ファイル一覧:
  - `<相対パス1>`
  - `<相対パス2>`
  - ...
```

全層の記載が終わったら、末尾に合計採用件数（全層の k の総和）を記す。Phase 2 の
完了条件（全層合計20ファイル以上）は、この合計値で判定する。

## 禁止事項

- 乱数・`shuf`・タイムスタンプ順など非決定的な選定手順を使わない
- 層内から「代表的に見える」ファイルを目視で選ぶことを禁止する（決定的コマンドの
  出力のみを根拠にする）
- サンプル記録に書いたコマンドと、実際に Phase 3〜4 で参照したファイル集合が
  食い違うことを禁止する（記録後に対象ファイルを追加・差し替えたら記録も更新する）
