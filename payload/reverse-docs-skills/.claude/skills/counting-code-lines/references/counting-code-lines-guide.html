コード行数計測スキル
counting-code-lines

対象ディレクトリのコード行数・ファイル数を FE/BE 別に計測し、code-metrics.json を機械的に生成するスキルの確定仕様。

§1概要

本スキルは、対象リポジトリのソースコード行数（total／FE／BE 別）・ファイル数・テスト件数を計測し、code-metrics.json を生成する。env-config.json（surveying-local-environment の出力）を参照して cloc の有無を判定し、cloc がインストール済みならそちらを優先、不在なら find + wc -l によるフォールバック計測に切り替える。

テストの計数は 2 段階で行う。まずテストファイル（*.test.ts / *.spec.ts / test_*.py 等）を列挙し、次にその内容から it( / test( / def test_ の出現数を grep で数える。テストファイルを先に絞り込んでから計数することで、テスト以外のファイル内の偶発的な文字列一致を防ぐ。

── 処理の全体像 ──────────────────────────────

  target_dir（計測対象ディレクトリ）
       │  Phase 1  前提確認（target_dir 存在確認・env_config 有無確認）
       ▼
  env-config.json（tools.cloc の有無）
       │  Phase 2  計測方式の決定（cloc 方式 / wc -l 方式）
       ▼
       │  Phase 3  コード行数の計測
       │  Phase 4  FE/BE 分離（パスパターン判定）
       │  Phase 5  テスト計測（テストファイル列挙 → it/test/def test_ を計数）
       ▼
  code-metrics.json（Phase 6）

§2出力

スキルを実行すると、以下の 1 点が生まれる。

code-metrics.json（output_dir 直下）
コード行数の total／fe／be、ファイル数の file_count（fe_files／be_files 内訳付き）、計測方式を示す method（"cloc" または "wc"）、計測時刻 measured_at、計測時のコミットハッシュ commit（git 管理外なら null）、テスト件数と FE/BE・ファイル数の内訳を持つ tests、直前計測との比較用 previous（初回計測時は null）を持つ構造化 JSON。読んだ人はこの 1 ファイルだけで、コード規模がどの計測方式でいつ測られ、前回計測からどう推移したかを追跡できる。
