# フォルダ構成のベストプラクティス

スキルフォルダの正しい構成と、references/ フォルダの適切な使い方を解説します。

---

## 標準的なスキルフォルダ構成

```
my-skill/
├── SKILL.md              # 必須: コア仕様（500行以下推奨）
├── references/           # 推奨: そのスキル固有の詳細情報
│   ├── patterns.md       # パターン集
│   ├── examples.md       # 詳細な例
│   └── advanced.md       # 高度な使い方
├── scripts/              # オプション: 実行可能スクリプト
│   └── run-task.sh
└── assets/               # オプション: 出力用テンプレート
    └── template.json
```

---

## 各ディレクトリの役割

| ディレクトリ | 役割 | 置くべきもの | 置いてはいけないもの |
|-------------|------|-------------|---------------------|
| `SKILL.md` | コア仕様 | 概要、使用タイミング、基本ワークフロー | 詳細な例、長い説明 |
| `references/` | **そのスキル固有**の詳細情報 | パターン集、詳細例、高度な使い方 | 公式ドキュメントのコピー |
| `scripts/` | 実行可能スクリプト | .sh, .py ファイル | ドキュメント |
| `assets/` | 出力用テンプレート | テンプレートファイル | スクリプト |

---

## references/ フォルダの正しい使い方

### 置くべきもの

✅ **そのスキル固有の詳細情報**
- パターン集（そのスキルで使うパターン）
- 詳細な例（コア仕様に入りきらない例）
- 高度な使い方（上級者向け情報）
- トラブルシューティング（そのスキル固有の問題解決）

### 置いてはいけないもの

❌ **公式ドキュメントのコピー**
- `/docs/skills.md` をコピーしてはいけない
- Claude Code 全体の仕様書を複製してはいけない

❌ **他のスキルと重複する内容**
- 汎用的なベストプラクティス
- 複数スキルで共通する情報

❌ **全体共通の仕様書**
- 全体共通の情報は `/docs/` に配置する

## 正しい例と間違った例

### 正しい例

```
creating-skills/
├── SKILL.md                      # コア仕様
└── references/
    ├── folder-structure.md       # このスキル固有: フォルダ構成の詳細
    ├── description-examples.md   # このスキル固有: description の例集
    └── anti-patterns.md          # このスキル固有: アンチパターン集
```

**理由**: references/ 内のファイルは全て「スキル作成ガイド」固有の詳細情報

### 間違った例

```
creating-skills/
├── SKILL.md
└── references/
    └── skills.md                 # ❌ /docs/skills.md のコピー
```

**問題点**:
- `/docs/skills.md` と内容が重複
- 更新時に2箇所を同期する必要がある
- references/ の役割を誤解している

---

## 公式ドキュメントへの正しい参照方法

公式ドキュメントはコピーせず、参照リンクで済ませる:

```markdown
## 参照資料

- Claude Skills の全体仕様: `/docs/skills.md`
- Progressive Disclosure の詳細: `/docs/skills.md#progressive-disclosure`
```

---

## Progressive Disclosure を活かす構成

### Stage 2 (SKILL.md) に入れるもの
- 概要（1-2段落）
- 使用タイミング
- 基本ワークフロー
- 最も重要なルール

### Stage 3 (references/) に分離するもの
- 詳細な例（10個以上の例など）
- 高度な使い方
- トラブルシューティング
- パターン集

**目安**: SKILL.md 本体は 500行/5000トークン以下に抑える

## チェックリスト

スキル作成時に確認:

- [ ] SKILL.md が 500行以下か
- [ ] references/ には公式ドキュメントのコピーがないか
- [ ] references/ 内は全てそのスキル固有の内容か
- [ ] 汎用的な情報は `/docs/` にあるか
- [ ] 公式ドキュメントは参照リンクで済ませているか
