# ワークフローガイド

> Category: 業務自動化
> Type: 手順型

複数の関連skillsの連携を、人が読んでわかるドキュメントとして <project>/skills/workflows/ に作成する。

## 使用タイミング

- 2つ以上の関連skillsを追加・整理した時
- スキル間の連携・使用順序を説明したい時
- 「ワークフローを作って」「どう使い分ける？」と聞かれた時

---

## 作成手順

### Step 1: 情報収集

ドキュメント作成前に以下を確認する。**情報が不足していればユーザーに質問して確認する。**

1. **日本語ワークフロー名** — 何をするワークフローか一言で表す名前
   （例:「Gitコミット・PRフロー」「バグ調査〜修正フロー」）
2. **目的** — このワークフローで何を達成するか
3. **関連skills** — 使用するskillsと各々の役割
4. **トリガー** — どんな場面で使うか
5. **前提条件** — 事前に必要な状態・準備

### Step 2: ファイル作成

**配置場所:**
```
<project>/skills/workflows/<kebab-case-name>.md
```

**命名規則:** ワークフロー名をケバブケースに変換
- 「Gitコミット・PRフロー」→ `git-commit-pr.md`
- 「バグ調査〜修正フロー」→ `debug-fix.md`

### Step 3: 以下のテンプレートで作成

> **注意:** `updated:` には今日の日付（YYYY-MM-DD形式）を自動入力する。`skills:` には関連スキル名をすべて列挙する。

---

## ワークフロードキュメント テンプレート

```markdown
---
title: <日本語ワークフロー名>
skills: [skill-a, skill-b]
updated: YYYY-MM-DD
---

# <日本語ワークフロー名>

## 目的

<このワークフローで達成することを2〜3文で説明>

## トリガー

このワークフローを使う場面：
- <場面1>
- <場面2>

## 前提条件

- <必要な状態・準備1>
- <必要な状態・準備2>

## 関連スキル

| スキル | 役割 | 必須/任意 |
|-------|------|---------|
| `/skill-a` | <役割の説明> | 必須 |
| `/skill-b` | <役割の説明> | 必須 |
| `/skill-c` | <役割の説明> | 任意 |

## フロー図

\`\`\`mermaid
flowchart TD
    Start([開始]) --> A["skill-a<br/>ステップの説明"]
    A --> B{判断条件}
    B -->|条件A| C["skill-b<br/>ステップの説明"]
    B -->|条件B| D["skill-c<br/>ステップの説明"]
    C --> End([完了])
    D --> End
\`\`\`

## 各ステップの詳細

### 1. /skill-a — <役割>

**何をするか:** <説明>
**入力:** <必要な情報・状態>
**出力:** <完了後の状態>

### 2. /skill-b — <役割>

**何をするか:** <説明>
**入力:** <必要な情報・状態>
**出力:** <完了後の状態>

## 使用例

\`\`\`
# 例: <具体的なシナリオ>
1. <操作1>
2. <操作2>
3. <操作3>
\`\`\`

## 注意事項

- <注意点1>
- <注意点2>
```

---

## Mermaidノード記法リファレンス

| 記法 | 形状 | 用途 |
|-----|------|------|
| `[テキスト]` | 四角 | 処理・スキル呼び出し |
| `{テキスト}` | ひし形 | 分岐・判断 |
| `([テキスト])` | 角丸 | 開始・終了 |
| `["テキスト<br/>補足"]` | 四角・改行あり | スキル名＋説明を同時表示 |
| `-->` | 実線矢印 | 通常の遷移 |
| `-.->` | 破線矢印 | 任意・オプション |
| `-->|ラベル|` | ラベル付き矢印 | 条件・状態を明示 |

## 作成後チェックリスト

- [ ] 日本語ワークフロー名がわかりやすいか
- [ ] 目的が2〜3文で明確に書かれているか
- [ ] トリガー（使う場面）が列挙されているか
- [ ] 前提条件が記載されているか
- [ ] frontmatterの `skills:` リストと関連スキル表が一致しているか
- [ ] 関連スキルの役割・必須/任意が記載されているか
- [ ] Mermaid図で全スキルの流れが追えるか
- [ ] 各ステップに入力・出力が記載されているか
- [ ] 使用例が具体的か
- [ ] 注意事項が記載されているか
- [ ] 絶対パス・個人情報・社内情報が含まれていないか（公開安全ルール）

## Gotchas

- 既存ワークフロー文書を上書きする前に必ず Read で確認 — 追記か全書き直しかを判断してから Edit する
